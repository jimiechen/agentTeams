## 常见启动报错速查

**`Missing required env var: LARK_APP_ID`**。说明 `dotenv` 没读到 `.env`，确认 `.env` 在项目根目录（跟 `package.json` 同级），不是在 `src/` 下；另外 `tsx` 启动时必须在项目根目录执行 `npm run dev`，不能 `cd src && tsx index.ts`。

**`Error: No suitable Trae target found`**。Trae 没启动调试模式，或者端口被别的 Chrome 占了。`lsof -iTCP:9222` 看看是谁在监听，如果是普通 Chrome 实例就 `--remote-debugging-port=9223` 换端口，同步改 `.env` 的 `CDP_PORT`。

**`[selectors] primary failed for chat.input, using fallback[0]`**。这是警告不是错误，说明 `[contenteditable='true'][class*='chat-input']` 这个选择器没命中，fallback 到了 `[contenteditable='true']`。如果 `[contenteditable='true']` 也命中了，功能正常；但建议用 Trae 开发者工具看一下真实 DOM，把正确的 selector 回填到 `config/selectors.v2026-04-26.json` 的 `chat.input.primary`。

**飞书 WS 能连上但收不到消息**。99% 是飞书开发者后台的权限或事件订阅没发布。用 `DEBUG=@larksuiteoapi/*,mvp:*` 能看到更详细的 WS 握手日志，如果握手 200 但一直没事件，去后台点 "版本管理 → 创建版本 → 申请发布"。

**`fillPrompt verification failed`**。`innerText` 注入后回读不一致。可能 Trae 用了受控组件强制覆盖了 `innerText`。临时兜底方案：把 `src/actions/fill-prompt.ts` 里的 `innerText = X; dispatchEvent('input')` 改成 `document.execCommand('insertText', false, X)`——这是 contenteditable 最兼容的写法，且支持后续拟人化改造（逐字符调用）。

**`waitResponse timeout after 90000ms`**。AI 响应 90 秒没生成完，要么 Prompt 太长要么 Trae 侧抽风。把 `config/pmbot.yaml` 的 `response_timeout_ms` 调到 `180000` 即 3 分钟，或者把 `waitResponse` 的 `stableMs` 从 1500 调到 2500（减少"生成慢导致误判还没完成"的概率）。
