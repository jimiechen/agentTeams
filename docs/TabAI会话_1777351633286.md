### **行号错位是 Trae Chat DOM 的 Markdown 容器被按行切分渲染后，textContent 抓取时把每行的行号、缩进、代码块边框全都拼接进来导致的——解法不是"转 Markdown"，而是在回贴飞书前做一次"Trae DOM → Markdown"的**结构化重建**，同时用飞书的富文本卡片消息而不是纯文本消息投递**

你遇到的这个问题非常典型，本质上是**三层渲染错配**：Trae 用富文本组件渲染 Markdown → 你用 `textContent` 拉成字符串 → 飞书又把字符串当纯文本渲染。三次渲染一叠加，代码块的行号、语法高亮的每个 token 的空格、表格的分隔符全部错位。下面给你一套可以直接合入的工程方案。

## **一、错位现象的根因拆解**

先看当前抓取链路上三层发生了什么。Trae Chat 里一个包含代码块的 AI 回复，DOM 大致长这样：

```html
<div class="chat-turn">
  <div class="markdown-body">
    <p>这是一段说明</p>
    <div class="code-block-wrapper">
      <div class="code-header">
        <span class="language">python</span>
        <button class="copy-btn">复制代码</button>
      </div>
      <pre class="code-content">
        <div class="line"><span class="line-num">1</span><span class="code">def foo():</span></div>
        <div class="line"><span class="line-num">2</span><span class="code">    return 1</span></div>
      </pre>
    </div>
  </div>
</div>
```

当前 `wait-response.ts` 做的事情是 `lastTurn.textContent` 或 `innerText`，结果是：

```
这是一段说明
python复制代码
1def foo():2    return 1
```

**行号和代码黏在一起、语言标签和复制按钮黏在一起、空格被吞、换行被吞**——这就是你在飞书群里看到的"行号错位无法阅读"的真相。这不是飞书的问题，也不是 Markdown 的问题，是**你在 textContent 这一层丢失了所有结构信息**。

## **二、解法方向：不要转 Markdown，要"重建 Markdown"**

"转 Markdown" 这个词容易产生误导，让人以为装个 `turndown` 库把 HTML 转一下就完事。实际行不通，因为 Trae 的 DOM 已经不是标准 HTML 语义——`<div class="line">` 承担了 `<pre>` 的角色、`<span class="line-num">` 是渲染噪声不是内容。`turndown` 这类通用 HTML→MD 工具会把噪声也当内容转进来。

**正确做法是"在 Trae DOM 里按 Trae 的约定重建 Markdown"**——写一个只认识 Trae Chat DOM 结构的**定制序列化器**，它只做三件事：识别段落、识别代码块（去掉行号和工具栏）、识别表格与列表。100 行 JS 可以搞定，且因为只认 Trae 的 class，不会被通用工具的启发式误伤。

## **三、完整实现：`extractMarkdown()` 定制序列化器**

把下面这段加到 `src/actions/wait-response.ts` 里，替换原来那个 `lastTurn.textContent` 的抓取：

```typescript
async function extractMarkdownResponse(cdp: CDPClient): Promise<string> {
  const markdown = await cdp.evaluate<string>(`
    (() => {
      const turns = document.querySelectorAll('.chat-turn');
      const lastTurn = turns[turns.length - 1];
      if (!lastTurn) return '';

      // 只取 AI 消息的正文容器（排除工具栏、头像等）
      const body = lastTurn.querySelector('[class*="markdown-body"], [class*="message-content"]') || lastTurn;

      return serializeToMarkdown(body);

      function serializeToMarkdown(root) {
        const lines = [];
        walk(root, lines, 0);
        return lines.join('\\n').replace(/\\n{3,}/g, '\\n\\n').trim();
      }

      function walk(node, lines, depth) {
        if (!node) return;

        // 跳过工具栏、复制按钮、行号等噪声
        if (isNoise(node)) return;

        // Element 节点按 tag 分派
        if (node.nodeType === 1) {
          const tag = node.tagName.toLowerCase();
          const cls = typeof node.className === 'string' ? node.className : '';

          // 代码块：识别 Trae 的 code-block-wrapper / pre / .code-content
          if (cls.includes('code-block') || tag === 'pre') {
            const lang = extractLang(node);
            const code = extractCodeContent(node);
            if (code) {
              lines.push('');
              lines.push('\`\`\`' + (lang || ''));
              lines.push(code);
              lines.push('\`\`\`');
              lines.push('');
            }
            return;
          }

          // 行内代码
          if (tag === 'code') {
            lines.push('\`' + (node.textContent || '') + '\`');
            return;
          }

          // 标题
          if (/^h[1-6]$/.test(tag)) {
            const level = parseInt(tag[1], 10);
            lines.push('');
            lines.push('#'.repeat(level) + ' ' + (node.textContent || '').trim());
            lines.push('');
            return;
          }

          // 段落
          if (tag === 'p') {
            const text = collectInline(node);
            if (text.trim()) lines.push(text);
            lines.push('');
            return;
          }

          // 无序列表
          if (tag === 'ul') {
            node.querySelectorAll(':scope > li').forEach(li => {
              lines.push('- ' + collectInline(li));
            });
            lines.push('');
            return;
          }

          // 有序列表
          if (tag === 'ol') {
            Array.from(node.querySelectorAll(':scope > li')).forEach((li, i) => {
              lines.push((i + 1) + '. ' + collectInline(li));
            });
            lines.push('');
            return;
          }

          // 表格
          if (tag === 'table') {
            lines.push(serializeTable(node));
            lines.push('');
            return;
          }

          // 引用
          if (tag === 'blockquote') {
            const inner = collectInline(node).split('\\n').map(l => '> ' + l).join('\\n');
            lines.push(inner);
            lines.push('');
            return;
          }

          // 链接
          if (tag === 'a') {
            const href = node.getAttribute('href') || '';
            const text = (node.textContent || '').trim();
            lines.push('[' + text + '](' + href + ')');
            return;
          }

          // 图片
          if (tag === 'img') {
            const src = node.getAttribute('src') || '';
            const alt = node.getAttribute('alt') || 'image';
            lines.push('![' + alt + '](' + src + ')');
            return;
          }

          // 分隔线
          if (tag === 'hr') {
            lines.push('');
            lines.push('---');
            lines.push('');
            return;
          }

          // 加粗 / 斜体 等内联元素：递归
          if (['strong', 'b'].includes(tag)) {
            lines.push('**' + (node.textContent || '') + '**');
            return;
          }
          if (['em', 'i'].includes(tag)) {
            lines.push('*' + (node.textContent || '') + '*');
            return;
          }

          // 其他容器：递归子节点
          for (const child of node.childNodes) walk(child, lines, depth + 1);
          return;
        }

        // 文本节点
        if (node.nodeType === 3) {
          const text = node.textContent || '';
          if (text.trim()) lines.push(text);
        }
      }

      // 识别噪声节点：行号、工具栏按钮、复制提示、语言标签容器
      function isNoise(node) {
        if (node.nodeType !== 1) return false;
        const cls = typeof node.className === 'string' ? node.className : '';
        if (!cls) return false;
        const noisePatterns = [
          'line-number', 'line-num', 'linenumber',
          'code-header', 'code-toolbar', 'copy-btn', 'copy-button',
          'message-actions', 'toolbar', 'action-bar',
          'feedback', 'thumbs',
        ];
        return noisePatterns.some(p => cls.toLowerCase().includes(p));
      }

      // 提取代码块语言：从 class="language-python" 或头部 span 里
      function extractLang(codeBlock) {
        const byClass = (codeBlock.className || '').match(/language-(\\w+)/);
        if (byClass) return byClass[1];
        const langEl = codeBlock.querySelector('[class*="language"]:not([class*="language-"])');
        return (langEl?.textContent || '').trim().toLowerCase();
      }

      // 提取代码内容：按行组装，忽略行号 span
      function extractCodeContent(codeBlock) {
        const lines = codeBlock.querySelectorAll('[class*="line"]:not([class*="line-num"])');
        if (lines.length > 0) {
          // Trae 的按行 DOM 结构
          return Array.from(lines).map(ln => {
            // 移除每行内的行号 span
            const clone = ln.cloneNode(true);
            clone.querySelectorAll('[class*="line-num"], [class*="line-number"]').forEach(n => n.remove());
            return clone.textContent || '';
          }).join('\\n');
        }
        // Fallback：直接取 pre/code 的 textContent
        const pre = codeBlock.querySelector('pre, code');
        return (pre?.textContent || codeBlock.textContent || '').trim();
      }

      // 收集内联文本：处理 <code>、<strong>、<em> 等
      function collectInline(node) {
        const parts = [];
        for (const child of node.childNodes) {
          if (child.nodeType === 3) {
            parts.push(child.textContent || '');
          } else if (child.nodeType === 1) {
            if (isNoise(child)) continue;
            const tag = child.tagName.toLowerCase();
            if (tag === 'code') parts.push('\`' + child.textContent + '\`');
            else if (['strong', 'b'].includes(tag)) parts.push('**' + child.textContent + '**');
            else if (['em', 'i'].includes(tag)) parts.push('*' + child.textContent + '*');
            else if (tag === 'a') {
              parts.push('[' + child.textContent + '](' + (child.getAttribute('href') || '') + ')');
            } else {
              parts.push(collectInline(child));
            }
          }
        }
        return parts.join('').replace(/\\s+/g, ' ').trim();
      }

      // 简单表格序列化
      function serializeTable(table) {
        const rows = [];
        const headers = Array.from(table.querySelectorAll('thead th')).map(th => (th.textContent || '').trim());
        if (headers.length > 0) {
          rows.push('| ' + headers.join(' | ') + ' |');
          rows.push('| ' + headers.map(() => '---').join(' | ') + ' |');
        }
        table.querySelectorAll('tbody tr').forEach(tr => {
          const cells = Array.from(tr.querySelectorAll('td')).map(td => (td.textContent || '').trim());
          rows.push('| ' + cells.join(' | ') + ' |');
        });
        return rows.join('\\n');
      }
    })()
  `);
  return markdown || '';
}
```

这段代码做了四件事：**其一**，只处理 AI 消息正文容器（`markdown-body` / `message-content`），排除工具栏。**其二**，通过 `isNoise()` 黑名单识别并跳过行号、复制按钮、语言标签头等渲染噪声。**其三**，按 Trae DOM 的"按行 div"结构重建代码块内容，去掉每行的 `line-num span` 后再 `textContent`，行号错位问题根治。**其四**，把段落、标题、列表、表格、引用、链接、图片、加粗斜体全部按 Markdown 语法重建，不依赖通用 HTML 转 MD 库。

## **四、飞书回贴层：从 text 消息升级到 post（富文本）消息**

把 Markdown 文本直接塞进飞书的 `msg_type: "text"` 消息里，飞书也只会按纯文本渲染（换行保留但不会渲染代码块）。要让代码块真的显示成代码块、表格真的显示成表格，**必须用 `msg_type: "post"` 或 `msg_type: "interactive"`（卡片）**。

最简单的改造是 `post` 富文本，不需要设计卡片：

```typescript
// src/lark/client.ts 新增方法
async replyMarkdown(messageId: string, markdown: string): Promise<void> {
  // 把 Markdown 转成飞书 post 格式的 paragraphs
  const content = markdownToLarkPost(markdown);
  await this.sdk.im.message.reply({
    path: { message_id: messageId },
    data: {
      msg_type: 'post',
      content: JSON.stringify({
        zh_cn: {
          title: '',
          content,
        },
      }),
    },
  });
}

function markdownToLarkPost(md: string): Array<Array<{ tag: string; [k: string]: any }>> {
  const lines = md.split('\n');
  const paragraphs: Array<Array<any>> = [];
  let inCode = false;
  let codeBuffer: string[] = [];
  let codeLang = '';

  for (const line of lines) {
    // 代码块边界
    const codeStart = line.match(/^```(\w*)/);
    if (codeStart && !inCode) {
      inCode = true;
      codeLang = codeStart[1] || '';
      codeBuffer = [];
      continue;
    }
    if (line.startsWith('```') && inCode) {
      // 整块代码作为一个 text 段落输出，前后加语言标记
      paragraphs.push([{ tag: 'text', text: `【${codeLang || 'code'}】
${codeBuffer.join('\n')}`, style: ['bold'] }]);
      inCode = false;
      codeBuffer = [];
      continue;
    }
    if (inCode) {
      codeBuffer.push(line);
      continue;
    }

    // 空行
    if (!line.trim()) {
      paragraphs.push([{ tag: 'text', text: '' }]);
      continue;
    }

    // 链接识别
    const linkMatch = line.match(/^\[([^\]]+)\]\(([^)]+)\)$/);
    if (linkMatch) {
      paragraphs.push([{ tag: 'a', text: linkMatch[1], href: linkMatch[2] }]);
      continue;
    }

    // 普通文本（保留简单 Markdown 标记作为文本）
    paragraphs.push([{ tag: 'text', text: line }]);
  }
  return paragraphs;
}
```

**飞书 post 格式的好处**：代码块会以等宽字体显示、链接可点击、段落有正确换行、不会因为长内容被截断。缺点是 **post 格式不支持原生代码高亮**——如果要真正的代码高亮和 Markdown 渲染，必须升级到 **交互式卡片（interactive card）**，用 `markdown` 组件：

```typescript
async replyCard(messageId: string, markdown: string): Promise<void> {
  const card = {
    elements: [
      {
        tag: 'markdown',
        content: markdown,
      },
    ],
    header: {
      title: { tag: 'plain_text', content: 'AI 回复' },
    },
  };
  await this.sdk.im.message.reply({
    path: { message_id: messageId },
    data: {
      msg_type: 'interactive',
      content: JSON.stringify(card),
    },
  });
}
```

**飞书卡片的 markdown 组件支持**：标题、段落、列表、表格、代码块（含语言高亮）、引用、链接、图片——基本上就是完整 Markdown。这是最终形态。

## **六、长度与截断保护**

飞书单条消息有长度限制（text 5000 字符、post 大约 30KB、card markdown 30KB）。Trae 的大段回复（代码生成、长文档）很容易触发截断。建议在 `replyMarkdown` 里加一段分片策略：

```typescript
const MAX_LEN = 4000;
if (markdown.length <= MAX_LEN) {
  await replyOnce(markdown);
} else {
  // 按段落切分，避免切断代码块中间
  const chunks = splitByParagraph(markdown, MAX_LEN);
  for (let i = 0; i < chunks.length; i++) {
    await replyOnce(`[${i + 1}/${chunks.length}] ${chunks[i]}`);
    await sleep(300);  // 避免触发限流
  }
}
```

`splitByParagraph` 要保证不在 \`\`\`代码块\`\`\` 中间切断，不然分片后的每段代码块都不闭合。简单做法是只在空行处切分，并在切分后检测代码块边界数是否成对，不成对就回退到上一个安全切点。
