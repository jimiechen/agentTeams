## **二、三类失败的共性与差异（重新归纳）**

**共性（不变）**：三类都是任务进入了 **CDP 不主动推送事件的等待态**，光靠 `waitResponse` 的超时计时无法区分原因。纯等待超时的直接后果是任务被标记失败、用户看不到"到底卡在哪"。

**新增共性（脚本文档揭示的）**：**三类的探测信号都集中在同一组 CDP Runtime.evaluate 调用里**——只需要每 2 秒跑一次下面这段 JS，所有三类的状态都能一次性采齐：

```javascript

  btnIcon: document.querySelector('.chat-input-v2-send-button .codicon')?.className || '',
  btnDisabled: document.querySelector('.chat-input-v2-send-button')?.disabled || false,

    .find(b => /后台运行|取消/.test(b.textContent)),

  // 信号 4：删除文件弹窗（决定是否文件操作等待）
  hasDeleteCard: !!document.querySelector('.icd-delete-files-command-card-v2-actions-delete'),

  // 信号 5：侧边栏任务状态
  taskStatus: (() => {
    const active = document.querySelector('.index-module__task-item___zOpfg.selected, .index-module__task-item___zOpfg[class*="selected"]');
    const txt = active?.textContent || '';
    if (txt.includes('完成')) return 'completed';
    if (txt.includes('中断')) return 'interrupted';
    if (txt.includes('进行中')) return 'running';
    return 'unknown';
  })(),

  ts: Date.now(),
}))()

这比旧版 6 信号探针**更精简、更稳、且每个选择器都来自已锁版的脚本文档**，不是推测。

**差异（新识别出的优先级）**：弹窗 DOM（删除文件）的出现是**原子性**的——要么有要么没有，不会有过渡态；终端按钮（后台运行/取消）在任务结束瞬间会消失，有毫秒级抖动窗口；流式停滞则是一个**渐进判定**，要看滑动窗口内 textLen 的方差。所以优先级顺序必须是：**删除弹窗（原子）→ 终端按钮（次原子）→ 流式停滞（渐进）**，与画像表"诊断优先级"列一致。

// 等价于调 handle-terminal-timeout.js background
async function recoverTerminalHang(trae, policy = 'background') {
  const selectorText = policy === 'background' ? '后台运行' : '取消';
  return await trae.evaluate(`(() => {
    const buttons = document.querySelectorAll('.icd-btn.icd-btn-tertiary');
    const btn = Array.from(buttons).find(b => b.textContent.includes('${selectorText}'));
    if (btn && btn.offsetParent !== null) {
      btn.click();
      return { success: true, action: '${policy}' };
    }
    return { success: false, reason: 'button-not-visible' };
  })()`);
}
```

**恢复路径 B（删除弹窗）**：
```javascript
// 等价于调 handle-waiting-action.js keep
async function recoverDeleteModal(trae, policy = 'keep') {
  const selector = policy === 'delete'
    ? '.icd-delete-files-command-card-v2-actions-delete'
    : '.icd-delete-files-command-card-v2-actions-cancel';
  return await trae.evaluate(`(() => {
    const btn = document.querySelector('${selector}');
    if (btn) {
      btn.click();
      return { success: true, action: '${policy}' };
    }
    return { success: false, reason: 'modal-not-found' };
  })()`);
}
```

**恢复路径 C（模型停滞）**：
```javascript
// 等价于调 button-controller.stopGeneration
async function recoverModelStalled(trae) {
  const status = await trae.evaluate(`(() => {
    const btn = document.querySelector('.chat-input-v2-send-button');
    const icon = btn?.querySelector('.codicon');
    return { 
      isStop: (icon?.className || '').match(/stop/i) ? true : false,
      found: !!btn,
    };
  })()`);
  if (!status.isStop) return { success: false, reason: 'not-in-stop-state' };

  await trae.evaluate(`(() => {
    document.querySelector('.chat-input-v2-send-button').click();
    return true;
  })()`);
  return { success: true, action: 'stop' };
}
```

三条路径的**成功判定**都要在动作后等待 1~2 秒再抓一次快照验证信号消失：终端卡死恢复后 `hasTerminalBtn` 应变 `false`；弹窗恢复后 `hasDeleteCard` 应变 `false`；模型停滞恢复后 `btnIcon` 应从 `stop` 变 `ArrowUp`。信号没消失就视为恢复失败，进入回退策略。
