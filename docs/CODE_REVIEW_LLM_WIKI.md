| 亮点 | 说明 |

## 2. 关键设计决策

### 决策1: 三层压缩比 (24:1 → 11:1)

```
Layer 0 → Layer 1: 24:1 压缩 (8000字 → 500字)
Layer 1 → Layer 2: 11:1 压缩 (3500字 → 300字)
总压缩比: ~264:1
```

**优点**: 高度浓缩，信息密度极高  
**风险**: 可能丢失细节，需要验证五维度是否足够

**建议**: Phase 3验证期密切监控压缩质量

---

### 决策2: 工作区完全隔离

```
workspaces/PMCLI/wiki/  ←→  workspaces/DEVCLI/wiki/
      (完全独立)              (完全独立)
```

**优点**: 零污染，PMCLI的错误不影响DEVCLI  
**风险**: 通用知识需要重复存储

**建议**: 保留shared/目录用于跨工作区common知识

---

### 决策3: 五维度提炼模板

1. 新增知识
2. 失败模式
3. 用户偏好
4. 协议变更
5. 待办

**优点**: 结构化输出，便于检索和验证  
**风险**: 可能遗漏不属于这五个维度的重要信息

**建议**: 增加"其他重要信息"兜底维度

---

### 决策4: 1800字注入上限

```
Layer 2 Core:     300字 (固定)
Layer 1 Recent:  1500字 (最近3天)
Total:           1800字 (硬上限)
```

**优点**: 保证prompt长度可控  
**风险**: 对于复杂项目可能不够用

**建议**: 通过配置可调整，默认1800

## 3. 与现有系统的集成点

### 集成点1: runs/目录

现有结构:
```
runs/2026-04-28T10-00-00.md  (原始执行记录)
```

直接作为Layer 0输入，**无需改动**

---

### 集成点2: fillPrompt流程

现有流程:
```
fillPrompt(prompt) → 直接发送给AI
```

新流程:
```
fillPrompt(prompt) → wiki-inject skill → 注入背景知识 → 发送给AI
```

**改动**: 在fillPrompt前插入context building步骤

---

### 集成点3: 飞书消息

现有: 飞书消息触发任务执行，产生runs记录  
新增强: runs记录自动进入Layer 0 → 参与每日蒸馏

**无需改动现有飞书处理逻辑**

## 4. 待评审问题

### ❓ 问题1: 蒸馏调度时机

当前: 每天凌晨2点  
问题: 如果系统24小时运行，2点时可能有任务在执行

**选项**:
- A: 保持2点，假设任务已完成
- B: 改为任务空闲期检测
- C: 改为4点(低谷期)

---

### ❓ 问题2: 多工作区共享知识

当前: 完全隔离  
场景: PMCLI和DEVCLI共享的"CDP连接最佳实践"

**选项**:
- A: 保持完全隔离，各自维护
- B: 增加shared/目录存放common知识
- C: Layer 2支持import其他工作区知识

---

### ❓ 问题3: 记忆查询接口

当前: 通过 `/wiki query` 命令查询  
问题: 是否需要可视化界面?

**选项**:
- A: 命令行足够
- B: 增加飞书Bot查询功能
- C: 增加Web Dashboard

---

### ❓ 问题4: 蒸馏失败处理

场景: AI蒸馏服务不可用，或生成质量差  
当前: 文档未定义fallback策略

**建议**: 
- 蒸馏失败时保留昨日记忆
- 记录失败日志
- 人工介入告警

---

## 5. 实施风险矩阵

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 蒸馏质量不稳定 | 中 | 高 | 固定模板 + temperature=0.3 |
| 存储增长超预期 | 低 | 中 | 自动归档 + 保留策略 |
| 工作区记忆污染 | 低 | 高 | 目录隔离 + 独立进程 |
| LLM API成本 | 中 | 低 | 每日一次蒸馏，可控 |
| 提示词过长 | 低 | 中 | 1800字硬上限 |

## 6. 评审结论

### 推荐意见: ✅ 通过，建议Phase 1启动

**同意的设计**:
- 三层蒸馏架构清晰，压缩比合理
- 工作区隔离满足多Bot需求
- Skills + Rules实现方式符合Trae生态
- 与现有系统integration点明确

**建议修改**:
1. 增加"其他重要信息"作为第五维度的兜底
2. 明确shared/目录的共享机制
3. 定义蒸馏失败的fallback策略
4. 1800字上限改为配置项，默认1800

**建议补充**:
- 实际压缩比测算（用7天历史数据试运行）
- 蒸馏质量评估checklist
- 工作区记忆隔离的e2e测试
