# 2.5 心跳频率调优策略

## 默认频率配置

| 层级 | 默认频率 | 适用场景 |
|------|---------|---------|
| Layer 1 | 5秒 | 正常执行、健康状态 |
| Layer 2 | 15秒 | 正常执行、健康状态 |
| Layer 3 | 60秒 | 所有场景 |

## 动态调优策略

### 加速场景

**触发条件**：进入已知高危阶段

```javascript
// 检测到停滞风险
if (lastSignal.layer2.stalledSeconds > 20) {
    // Layer 1 从 5 秒降到 2 秒
    layer1Frequency = 2000;
    // Layer 2 从 15 秒降到 8 秒
    layer2Frequency = 8000;
}
```

**加速效果**：
- 更快发现异常
- 更快触发恢复动作
- 减少用户等待时间

**恢复机制**：异常解除后 60 秒回归默认频率

### 降频场景

**触发条件**：任务进入 queuing 状态

```javascript
// 检测到排队
if (lastSignal.layer2.isQueuing) {
    // Layer 1 从 5 秒升到 15 秒
    layer1Frequency = 15000;
    // Layer 2 从 15 秒升到 30 秒
    layer2Frequency = 30000;
}
```

**降频原因**：
- 排队时无法干预，频繁检测无意义
- 节省资源用于其他任务
- 避免不必要的 CDP 调用

**恢复机制**：排队解除后立即恢复默认频率

## 频率调优状态机

```
┌──────────────────┐     停滞风险>20s      ┌──────────────────┐
│   normal(默认)    │ ──────────────────→ │    accelerated    │
│  L1: 5s, L2:15s  │                     │  L1: 2s, L2:8s   │
└────────┬─────────┘                     └────────┬─────────┘
         │                                      │
         │ 60秒无异常                            │ 60秒无异常
         ←───────────────────────────────────────┘
         │
         ↓ 检测到排队
┌──────────────────┐
│    reduced       │
│  L1:15s, L2:30s  │
└────────┬─────────┘
         │
         │ 排队解除
         ←─────────────────────────────┐
                                      │
                                      ↓ (从其他状态转入)
```

## 调优参数配置

| 参数 | 默认值 | 说明 |
|------|-------|------|
| `accelerateThreshold` | 20秒 | 触发加速的停滞秒数 |
| `accelerateDuration` | 60秒 | 加速状态持续时间 |
| `acceleratedLayer1Freq` | 2秒 | 加速时 Layer 1 频率 |
| `acceleratedLayer2Freq` | 8秒 | 加速时 Layer 2 频率 |
| `reducedLayer1Freq` | 15秒 | 降频时 Layer 1 频率 |
| `reducedLayer2Freq` | 30秒 | 降频时 Layer 2 频率 |

## 资源消耗对比

| 场景 | Layer 1 CPU | Layer 2 CPU | 总 CPU |
|------|-----------|-----------|--------|
| 默认 | ~0.2% | ~0.1% | ~0.3% |
| 加速 | ~0.5% | ~0.15% | ~0.65% |
| 降频 | ~0.07% | ~0.03% | ~0.1% |

## 调优效果评估

### 加速场景收益
- 异常发现时间缩短 60%
- 恢复响应时间缩短 60%
- 用户等待感知减少

### 降频场景收益
- CPU 占用降低 67%
- CDP 调用减少 67%
- 网络开销降低 67%

## 特殊情况处理

### 任务完成
所有层级频率恢复默认，等待下次任务启动。

### 任务中断
所有层级停止，等待人工介入。

### CDP 连接断开
保持当前频率配置，重连后继续执行。
