# 2.4 三层之间的信号传递与状态复用

## 架构设计

三层心跳通过**共享内存**实现信号传递，避免重复采集：

```
┌─────────────────────────────────────────────────────────────┐
│  Layer 1 (5s)                                              │
│  产出：taskStatus + 4个按钮 boolean                         │
│  写入共享内存 lastSignal.layer1                             │
└─────────────────────────────────────────────────────────────┘
              │ 供 Layer 3 复用
              ▼
┌─────────────────────────────────────────────────────────────┐
│  Layer 2 (15s)                                             │
│  产出：charsDelta + stalledSeconds + isQueuing              │
│  更新共享内存 lastSignal.layer2                             │
└─────────────────────────────────────────────────────────────┘
              │ 供 Layer 3 复用
              ▼
┌─────────────────────────────────────────────────────────────┐
│  Layer 3 (60s)                                             │
│  读取 lastSignal.layer1 + lastSignal.layer2                │
│  + 自己采集的深度指标                                       │
│  合并为完整 HeartbeatSignal 写入 jsonl                      │
└─────────────────────────────────────────────────────────────┘
```

## 共享内存结构

```javascript
const lastSignal = {
    layer1: {
        timestamp: 0,
        taskStatus: 'unknown',
        hasBackgroundButton: false,
        hasCancelButton: false,
        hasKeepButton: false,
        hasDeleteButton: false
    },
    layer2: {
        timestamp: 0,
        textLength: 0,
        charsDelta: 0,
        isQueuing: false,
        queuePosition: null,
        stalledSeconds: 0
    },
    layer3: {
        timestamp: 0,
        buttonSnapshot: [],
        sessionDuration: 0,
        totalChars: 0,
        contextUsage: 0,
        lastRecoveryTime: null,
        heartbeatCount: 0
    },
    currentMode: 'healthy',
    recoveryAttempts: {
        terminalHang: 0,
        modalBlocking: 0,
        modelStalled: 0
    }
};
```

## 信号传递流程

### Layer 1 → Layer 3
```javascript
// Layer 3 读取 Layer 1 信号
const { taskStatus, hasBackgroundButton } = lastSignal.layer1;
```

### Layer 2 → Layer 3
```javascript
// Layer 3 读取 Layer 2 信号
const { isQueuing, stalledSeconds } = lastSignal.layer2;
```

### 统一快照
Layer 3 在写入 jsonl 时，读取的是同一时刻的 Layer 1/2 快照，保证数据一致性。

## 状态复用场景

### 场景 1：模式判定
```javascript
// 使用 Layer 1 + Layer 2 信号进行模式判定
function determineMode() {
    const { taskStatus, hasBackgroundButton } = lastSignal.layer1;
    const { isQueuing, stalledSeconds } = lastSignal.layer2;
    
    // 优先级判定逻辑
    if (taskStatus === 'interrupted') return 'task-interrupted';
    if (hasBackgroundButton) return 'terminal-hang';
    if (isQueuing) return 'model-queuing';
    if (stalledSeconds >= 30) return 'model-stalled';
    return 'healthy';
}
```

### 场景 2：恢复动作验证
```javascript
// 执行恢复动作后，验证 Layer 1 信号是否变化
async function verifyRecovery(mode) {
    await sleep(5000); // 等待 5 秒
    await layer1FastProbe(cdp); // 刷新 Layer 1 信号
    
    if (mode === 'terminal-hang') {
        return !lastSignal.layer1.hasBackgroundButton;
    }
    return true;
}
```

## 同步机制

### 读写锁
使用简单的标志位防止并发问题：

```javascript
let isWriting = false;

async function updateSignal(layer, data) {
    while (isWriting) {
        await sleep(10);
    }
    isWriting = true;
    lastSignal[layer] = { ...lastSignal[layer], ...data, timestamp: Date.now() };
    isWriting = false;
}
```

### 时间戳校验
每次读取时检查时间戳，避免使用过期数据：

```javascript
function getFreshSignal(layer) {
    const signal = lastSignal[layer];
    const maxAge = layer === 'layer1' ? 10000 : 30000; // Layer 1: 10秒, 其他: 30秒
    
    if (Date.now() - signal.timestamp > maxAge) {
        throw new Error(`${layer} signal is stale`);
    }
    return signal;
}
```

## 优势分析

### 性能提升
- 避免重复的 CDP 调用
- Layer 3 只需做一次深度采集
- 减少网络往返

### 数据一致性
- 同一时刻的完整快照
- 避免跨层数据冲突
- 便于问题追溯

### 可维护性
- 信号定义集中管理
- 各层职责清晰
- 易于扩展新的采集项
