# 10.3 24 小时稳定性测试

## 10.3.1 为什么需要长时间测试

E2E 测试验证**功能正确**，长时间测试验证**运行稳定**。两者关注不同问题：

| E2E 测试能发现 | 只有长时间测试能发现 |
|---------------|-------------------|
| 状态判定错误 | 内存泄漏 |
| 恢复动作失败 | Timer 累积漂移 |
| 通知格式错误 | 文件句柄耗尽 |
| 事件丢失 | WebSocket 断连未重连 |

心跳机制运行在长时任务的守护位置，任何资源泄漏都会在几小时后暴露。24 小时是**最低验证门槛**。

## 10.3.2 测试场景设计

24 小时测试分三个阶段，每个阶段 8 小时：

**阶段一（0-8h）：正常负载**

- 每 10 分钟提交一个真实任务（共 48 个）
- 不人工触发任何异常
- 验证心跳在空闲+任务交替时的资源占用

**阶段二（8-16h）：注入异常**

- 保持 10 分钟/任务节奏
- 每 30 分钟随机触发一次异常（五种模式均匀分布）
- 验证长时间运行下的恢复成功率是否稳定

**阶段三（16-24h）：高压场景**

- 缩短到 5 分钟/任务
- 同时运行 PMCLI 和 DEVCLI 两个工作区
- 每 15 分钟触发一次异常
- 验证高压下的心跳机制是否仍可靠

## 10.3.3 健康度指标与阈值

24 小时运行后，必须满足以下阈值：

| 指标 | 目标 | 测量方式 |
|------|------|---------|
| **内存增长** | < 50MB | `process.memoryUsage().rss` 的 24h 趋势 |
| **CPU 平均占用** | < 3% | 系统级监控（不含任务本身） |
| **心跳误判率** | < 0.5% | healthy 被误判为异常的次数 / 总心跳次数 |
| **恢复成功率** | > 85% | 触发恢复且回到 healthy 的比例 |
| **告警误报率** | < 5% | 被触发但事后被人工判定无需介入的比例 |
| **文件句柄泄漏** | = 0 | `lsof -p <pid>` 的 24h 趋势稳定 |
| **Timer 数量** | 稳定 | 不随时间无限增长 |

任何一项不达标都视为测试失败，必须修复后重跑。

## 10.3.4 数据采集工具

24 小时跑完后，需要批量分析 `heartbeat.jsonl`。专用脚本：

```typescript
// test-tools/analyze-24h-run.ts

import { readFileSync } from 'node:fs';

interface HeartbeatEntry {
  timestamp: number;
  workspace: string;
  layer: 1 | 2 | 3;
  mode: string;
  recoveryAction?: string;
  recoverySuccess?: boolean;
  signals: Record<string, any>;
}

function analyze(jsonlPath: string) {
  const entries: HeartbeatEntry[] = readFileSync(jsonlPath, 'utf-8')
    .trim().split('\n').map(l => JSON.parse(l));
  
  const totalHours = (entries.at(-1).timestamp - entries[0].timestamp) / 3600_000;
  
  return {
    totalHours,
    totalHeartbeats: entries.length,
    heartbeatsPerMinute: entries.length / totalHours / 60,
    modeDistribution: countBy(entries, 'mode'),
    recoveryAttempts: entries.filter(e => e.recoveryAction).length,
    recoverySuccessRate: entries.filter(e => e.recoverySuccess === true).length / 
                         entries.filter(e => e.recoveryAction).length,
    misclassifications: detectMisclassifications(entries),
  };
}

function detectMisclassifications(entries: HeartbeatEntry[]): number {
  let count = 0;
  for (let i = 1; i < entries.length - 1; i++) {
    const prev = entries[i - 1];
    const curr = entries[i];
    const next = entries[i + 1];
    
    if (prev.mode === 'healthy' && next.mode === 'healthy' && 
        curr.mode !== 'healthy' &&
        next.timestamp - curr.timestamp < 5000) {
      count++;
    }
  }
  return count;
}
```

运行命令：

```bash
node test-tools/analyze-24h-run.ts workspaces/PMCLI-test/runs/heartbeat.jsonl
```

输出示例：
