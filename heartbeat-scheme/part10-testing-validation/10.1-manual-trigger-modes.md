# 10.1 五种中断模式的人工触发方式

## 10.1.1 为什么需要人工触发

心跳机制的正确性验证面临一个根本矛盾：**异常是罕见事件**。等真实 Trae 异常自然发生来验证代码，可能等几天也不出现 `model-stalled`。开发阶段必须有**可复现的触发手段**，让五种中断模式按需发生。

## 10.1.2 terminal-hang 的触发

**原理**：让 Trae 的终端执行一个长时间不返回的命令，触发"后台运行"按钮。

**触发方式**：在 Trae 任务中输入 prompt：

```
请帮我执行 `sleep 300` 并等待结果
```

Trae 会调用终端工具执行 `sleep 300`，300 秒后自动出现"后台运行"和"取消"按钮。这是最稳定的触发方式，不依赖任何 mock。

**加速版本**：如果不想等 300 秒，可以在测试环境里改 Trae 的 terminal-timeout 阈值到 10 秒（需要 Trae 内部配置权限）。没有该权限时用真实 `sleep 300`。

## 10.1.3 modal-blocking 的触发

**原理**：让 Trae 完成一次写文件任务后，自动弹出"保留/删除"选择框。

**触发方式**：

```
请创建一个测试文件 test-modal.txt，内容为 "hello"。任务完成后不要清理。
```

任务完成后 Trae 会弹出"保留/删除"对话框等待用户选择。触发率 100%，且每次测试后记得手动选"删除"恢复环境。

## 10.1.4 model-stalled 的触发

**原理**：`model-stalled` 是隐式异常（文本 30 秒不增长），自然发生概率低。需要通过 **CDP 注入** mock。

**触发方式**：在测试专用的 mock 脚本里，用 CDP 修改 DOM 让最后一条 AI 消息的 `innerText` 停止更新：

```javascript
// test-tools/mock-stalled.js
async function triggerStalled(trae, durationMs = 40000) {
  const turnIndex = await trae.evaluate(`
    document.querySelectorAll('.chat-turn').length - 1
  `);
  
  await trae.evaluate(`
    (() => {
      const turn = document.querySelectorAll('.chat-turn')[${turnIndex}];
      const body = turn.querySelector('[class*="markdown-body"]');
      
      window.__frozenContent = body.innerText;
      
      Object.defineProperty(body, 'innerText', {
        get: () => window.__frozenContent,
        configurable: true
      });
    })()
  `);
  
  console.log(`✅ 已冻结 AI 消息 #${turnIndex}，将在 ${durationMs}ms 后解冻`);
  
  setTimeout(async () => {
    await trae.evaluate(`
      (() => {
        const turn = document.querySelectorAll('.chat-turn')[${turnIndex}];
        const body = turn.querySelector('[class*="markdown-body"]');
        delete body.innerText;
      })()
    `);
    console.log('✅ 已解冻');
  }, durationMs);
}
```

**验证点**：Layer 2 应在 30 秒内（2 次 × 15 秒）检测到 `charsDelta == 0`，判定为 `model-stalled`，触发 stop+retry。

## 10.1.5 model-queuing 的触发

**原理**：让 Trae 最后一条消息的文本包含"排在第 X 位"字样。

**触发方式**：CDP 注入文本：

```javascript
async function triggerQueuing(trae, position = 5) {
  await trae.evaluate(`
    (() => {
      const turns = document.querySelectorAll('.chat-turn');
      const lastTurn = turns[turns.length - 1];
      const body = lastTurn.querySelector('[class*="markdown-body"]') || lastTurn;
      
      const queueHint = document.createElement('div');
      queueHint.textContent = '请求量较高，排在第 ${position} 位，请稍候...';
      queueHint.id = 'mock-queue-hint';
      body.appendChild(queueHint);
    })()
  `);
}

async function clearQueuing(trae) {
  await trae.evaluate(`
    document.querySelector('#mock-queue-hint')?.remove();
  `);
}
```

**验证点**：Layer 2 检测到"排在第"字样后，`mode` 应判定为 `model-queuing`，**不触发恢复动作**（这是正确行为，排队不是异常）。

## 10.1.6 task-interrupted 的触发

**原理**：让任务列表里的 PMCLI 任务状态文本变成"中断"。

**触发方式**：CDP 注入：

```javascript
async function triggerInterrupted(trae) {
  await trae.evaluate(`
    (() => {
      const items = document.querySelectorAll('.index-module__task-item___zOpfg');
      for (const item of items) {
        if (item.textContent.includes('PMCLI')) {
          const statusSpan = document.createElement('span');
          statusSpan.textContent = ' 中断';
          statusSpan.id = 'mock-interrupted';
          item.appendChild(statusSpan);
          break;
        }
      }
    })()
  `);
}
```

**验证点**：Layer 1 立即检测到，`mode` 判定为 `task-interrupted`，**不尝试恢复**，直接发送 critical 告警，任务机进入 `interrupted` 终态。

## 10.1.7 触发工具打包

以上五种触发方式打包到 `test-tools/heartbeat-triggers.js`，提供统一 CLI：

```bash
# 触发 terminal-hang（需要配合任务执行）
node test-tools/heartbeat-triggers.js terminal-hang

# 触发 model-stalled，持续 40 秒
node test-tools/heartbeat-triggers.js stalled --duration=40000

# 触发 model-queuing，排在第 5 位
node test-tools/heartbeat-triggers.js queuing --position=5

# 触发 task-interrupted
node test-tools/heartbeat-triggers.js interrupted

# 清理所有 mock 状态
node test-tools/heartbeat-triggers.js cleanup
```

这套工具是所有后续测试用例的基础——没有它，自动化测试几乎不可能。
