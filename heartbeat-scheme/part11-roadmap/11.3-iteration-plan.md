# 11.3 优化迭代与长期演进

## 11.3.1 优化方向概览

完整版上线后，从三个方向持续优化：

| 方向 | 目标 | 时间框架 |
|------|------|---------|
| 策略自适应 | 恢复策略根据场景自动调整 | 中期（1-2 月） |

## 11.3.2 策略自适应

### 问题背景

固定恢复策略存在局限：

- modal-blocking：默认"保留"可能堆积无用文件
- model-stalled：stop+retry 可能丢失上下文
- terminal-hang：某些场景需要"取消"而非"后台运行"

### 自适应策略设计

```typescript
interface AdaptiveStrategy {
  mode: InterruptMode;
  context: TaskContext;
  history: RecoveryHistory[];
  
  decide(): RecoveryAction;
}

interface TaskContext {
  taskType: 'code_change' | 'file_operation' | 'analysis' | 'unknown';
  fileCount: number;
  hasUncommittedChanges: boolean;
  previousRecoveries: number;
}

interface RecoveryHistory {
  mode: InterruptMode;
  action: RecoveryAction;
  result: 'success' | 'failed' | 'partial';
  timestamp: number;
}
```

### 自适应规则示例

**modal-blocking 自适应**：

```typescript
function decideModalAction(context: TaskContext): 'keep' | 'delete' {
  if (context.taskType === 'file_operation' && context.fileCount > 10) {
    return 'delete';
  }
  if (context.hasUncommittedChanges) {
    return 'keep';
  }
  if (context.previousRecoveries > 3) {
    return 'delete';
  }
  return 'keep';
}
```

**terminal-hang 自适应**：

```typescript
function decideTerminalAction(context: TaskContext): 'background' | 'cancel' {
  if (context.taskType === 'analysis') {
    return 'cancel';
  }
  if (context.previousRecoveries > 2) {
    return 'cancel';
  }
  return 'background';
}
```

### 实施路径

| 阶段 | 内容 | 时间 |
|------|------|------|
| 数据收集 | 记录每次恢复的上下文和结果 | 2 周 |
| 规则提取 | 分析数据，提取自适应规则 | 2 周 |
| 规则实现 | 实现自适应决策逻辑 | 1 周 |
| A/B 验证 | 对比固定策略 vs 自适应策略 | 2 周 |

## 11.3.4 知识沉淀

### 沉淀目标

1. **恢复案例库**：记录每次恢复的完整上下文
2. **策略知识库**：沉淀有效的恢复策略
3. **问题模式库**：归纳常见问题模式

### 恢复案例库

每个恢复案例记录：

```yaml
case_id: case-20240115-001
timestamp: 2024-01-15T10:30:00Z
mode: terminal-hang
task_context:
  task_type: code_change
  task_id: task-123
  prompt: "请帮我重构 utils.ts 文件"
detection:
  layer: layer1
  confidence: 0.95
  duration_ms: 50
recovery:
  action: click_background_run
  result: success
  duration_ms: 200
follow_up:
  task_completed: true
  user_feedback: null
```

### 策略知识库

从案例中提取的策略：

```yaml
strategy_id: strategy-terminal-hang-001
mode: terminal-hang
conditions:
  - task_type == "code_change"
  - previous_recoveries < 2
action: click_background_run
success_rate: 0.92
sample_count: 50
last_updated: 2024-01-15
```

### 问题模式库

归纳的问题模式：

```yaml
pattern_id: pattern-001
name: "长时间编译导致 terminal-hang"
description: "执行 npm run build 超过 60 秒触发 terminal-hang"
frequency: high
affected_tasks:
  - code_change
  - file_operation
recommended_action: click_background_run
prevention:
  - 建议: "拆分大型构建任务"
  - 建议: "使用增量编译"
```

### 知识应用

1. **新任务预警**：根据任务特征预测可能的中断模式
2. **策略推荐**：根据历史数据推荐最优恢复策略
3. **问题预防**：提前提示用户可能的问题

## 11.3.6 风险与止损

### 长期风险

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 自适应策略误判 | 中 | 恢复效果变差 | 保留固定策略 fallback |
| 知识库污染 | 低 | 错误策略传播 | 人工审核机制 |
| 过度依赖自动恢复 | 中 | 用户技能退化 | 定期人工演练 |
