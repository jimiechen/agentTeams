# 4.3 Layer 2 内容探测脚本

## 设计目标

只扫描最后一条消息，成本比 Layer 1 更低，专门检测文本增量变化。

## CDP 脚本

```javascript
(() => {
  const turns = document.querySelectorAll('.chat-turn');
  const lastTurn = turns[turns.length - 1];
  if (!lastTurn) return { lastMessageChars: 0, isQueuing: false, queuePosition: null };

  const text = lastTurn.innerText || '';
  const queueMatch = text.match(/排在第\s*(\d+)\s*位/);

  return {
    lastMessageChars: text.length,
    isQueuing: text.includes('排队') || text.includes('请求量较高') || text.includes('等待中'),
    queuePosition: queueMatch ? parseInt(queueMatch[1]) : null,
  };
})()
```

## 增量计算（TS 侧）

增量计算不在浏览器里完成，而是在 TypeScript 侧：

```typescript
async contentCheck(): Promise<Partial<HeartbeatSignal>> {
  const raw = await this.cdp.evaluate<any>(SCRIPT_LAYER2);
  const charsDelta = raw.lastMessageChars - this.lastCharsCheckpoint;
  this.lastCharsCheckpoint = raw.lastMessageChars;

  // 停滞累计（连续无增长）
  let stalledSeconds = 0;
  if (charsDelta === 0 && !raw.isQueuing) {
    if (this.stalledStartTime === 0) this.stalledStartTime = Date.now();
    stalledSeconds = Math.floor((Date.now() - this.stalledStartTime) / 1000);
  } else {
    this.stalledStartTime = 0; // 重置
  }

  return { ...raw, charsDelta, stalledSeconds };
}
```

## innerText vs textContent

**为什么用 innerText？**

| 特性 | innerText | textContent |
|------|-----------|-------------|
| 隐藏元素 | 跳过 | 包含 |
| CSS 样式 | 受影响 | 不受影响 |
| 视觉语义 | 更贴近用户看到的内容 | 纯文本节点 |

用 `innerText` 更符合"用户视觉看到的内容增长"的语义，避免把折叠的推理过程算进去。

## 排队识别

### 关键词匹配
- `排队`
- `请求量较高`
- `等待中`

### 位次提取
使用正则 `/排在第\s*(\d+)\s*位/` 提取排队位置。

## 返回字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `lastMessageChars` | number | 最后一条消息字符数 |
| `charsDelta` | number | 与上次相比的增量 |
| `isQueuing` | boolean | 是否处于排队状态 |
| `queuePosition` | number \| null | 排队位置（如果在排队） |
| `stalledSeconds` | number | 连续停滞秒数 |

## 停滞判定逻辑

```
条件：charsDelta === 0 且 !isQueuing
- 如果满足：累计停滞时间
- 如果不满足：重置停滞计时器
```
