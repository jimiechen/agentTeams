# 4.4 Layer 3 深度探测脚本

## 设计目标

只采集 Layer 1/2 之外的深度信号，避免重复采集。

## CDP 脚本

```javascript
(() => {
  const visibleButtons = [];
  const buttons = document.querySelectorAll('button');
  for (const btn of buttons) {
    const style = window.getComputedStyle(btn);
    if (style.display !== 'none' && style.visibility !== 'hidden') {
      visibleButtons.push({
        ariaLabel: btn.getAttribute('aria-label') || '',
        text: btn.textContent?.trim().slice(0, 20) || '',
      });
    }
  }
  return { visibleButtons };
})()
```

## TS 侧拼装

在 TypeScript 侧拼装完整的深度指标：

```typescript
async deepCheck(): Promise<Partial<HeartbeatSignal>> {
  const raw = await this.cdp.evaluate<any>(SCRIPT_LAYER3);
  return {
    ...raw,
    sessionDurationMs: Date.now() - this.sessionStartTime,
    totalResponseChars: this.totalResponseChars,
    estimatedTokens: Math.ceil(this.totalResponseChars / 2.5),
  };
}
```

## 按钮快照的意义

### 发现未知异常
每 60 秒记录一次所有可见按钮，一旦出现未被 Layer 1 覆盖的异常按钮（比如 Trae 升级后新增的弹窗），可以通过 `heartbeat.jsonl` 回溯发现。

### 指导迭代
根据发现的新按钮，指导后续迭代 Layer 1 的采集清单。

## Token 估算

使用经验公式：`estimatedTokens = Math.ceil(chars / 2.5)`

| 文本类型 | 字符/Token 比值 |
|---------|---------------|
| 中文 | ~2.5 |
| 英文 | ~4 |
| 混合 | ~2.5（保守估算） |

## 返回字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `visibleButtons` | Array | 所有可见按钮的快照 |
| `sessionDurationMs` | number | 会话持续时长（毫秒） |
| `totalResponseChars` | number | 累计响应字符数 |
| `estimatedTokens` | number | 估算的 Token 消耗 |

## 采样间隔

Layer 3 始终保持 60 秒间隔，即使 Layer 1/2 触发了恢复动作。

## 数据用途

### 可观测性
- 健康度仪表盘
- 资源消耗监控
- 性能分析

### 故障排查
- 回溯历史状态
- 分析异常模式
- 定位问题根因
