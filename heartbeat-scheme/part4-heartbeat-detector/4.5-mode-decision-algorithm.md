# 4.5 模式判定算法

## 判定逻辑

按 Part 3.3 的优先级链，单次函数调用返回唯一模式：

```typescript
determineMode(s: Partial<HeartbeatSignal>): HealthMode {
  // 按优先级从高到低，第一个命中即返回
  if (s.taskStatus === 'interrupted') return 'task-interrupted';
  if (s.hasKeepButton || s.hasDeleteButton) return 'modal-blocking';
  if (s.hasBackgroundButton) return 'terminal-hang';
  if (s.isQueuing) return 'model-queuing';
  if ((s.stalledSeconds ?? 0) >= 30) return 'model-stalled';
  if (s.taskStatus === 'completed') return 'completed';
  if (s.taskStatus === 'in_progress') return 'healthy';
  return 'unknown';
}
```

## 优先级链

```
1. task-interrupted   （终态优先）
2. modal-blocking     （弹窗优先）
3. terminal-hang      （终端优先）
4. model-queuing      （排队优先于停滞）
5. model-stalled      （停滞最后判定）
6. completed          （正常完成）
7. healthy            （正常执行）
8. unknown            （信号缺失）
```

## 健壮性保护

### 信号缺失处理
任何信号字段缺失时，对应的判定分支自然跳过：

```typescript
// 使用 ?? 处理缺失值
if ((s.stalledSeconds ?? 0) >= 30) return 'model-stalled';
```

### 短路求值
一旦某个条件匹配，立即返回，不会继续检查后面的条件。

### 兜底返回
最终兜底返回 `unknown`，由 Daemon 决定重试或跳过。

## 判定示例

### 示例 1：正常执行
```javascript
// 输入
{ taskStatus: 'in_progress', hasBackgroundButton: false, ... }

// 输出
'healthy'
```

### 示例 2：终端挂起
```javascript
// 输入
{ taskStatus: 'in_progress', hasBackgroundButton: true, ... }

// 输出
'terminal-hang'
```

### 示例 3：信号缺失
```javascript
// 输入
{ taskStatus: undefined, ... }

// 输出
'unknown'
```

## 与 HeartbeatSignal 的集成

```typescript
async heartbeat(deep: boolean): Promise<HeartbeatSignal> {
  const layer1 = await this.fastCheck();
  const layer2 = await this.contentCheck();
  let layer3: Partial<HeartbeatSignal> = {};
  
  if (deep) {
    layer3 = await this.deepCheck();
  }
  
  const signal = {
    ...layer1,
    ...layer2,
    ...layer3,
  } as HeartbeatSignal;
  
  signal.mode = this.determineMode(signal);
  return signal;
}
```
