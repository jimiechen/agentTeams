# 4.1 HeartbeatDetector 类结构与核心接口

## 概述

HeartbeatDetector 是三层心跳检测机制的核心类，负责协调 Layer 1（快速探测）、Layer 2（内容探测）和 Layer 3（深度探测）的执行流程，并根据检测结果判定当前页面的心跳模式。

## 类定义

```typescript
interface HeartbeatConfig {
  layer1Timeout: number;      // Layer 1 超时时间（毫秒），默认 1000ms
  layer2Timeout: number;      // Layer 2 超时时间（毫秒），默认 5000ms
  layer3Timeout: number;      // Layer 3 超时时间（毫秒），默认 15000ms
  retryAttempts: number;      // 重试次数，默认 3
  retryDelay: number;         // 重试延迟（毫秒），默认 1000ms
}

interface DetectionResult {
  mode: HeartbeatMode;        // 检测到的模式
  confidence: number;         // 置信度 0-1
  signals: Signal[];          // 收集到的信号列表
  timestamp: number;          // 检测时间戳
  layer: 1 | 2 | 3;           // 在哪一层得出结论
}

type HeartbeatMode = 
  | 'normal'           // 正常模式：页面活跃，心跳正常
  | 'idle'             // 空闲模式：页面无用户交互
  | 'background'       // 后台模式：页面在后台运行
  | 'frozen'           // 冻结模式：页面被浏览器冻结
  | 'crashed';         // 崩溃模式：页面已崩溃

class HeartbeatDetector {
  private config: HeartbeatConfig;
  private currentMode: HeartbeatMode;
  private signalBuffer: Signal[];
  private layer1Timer: NodeJS.Timeout | null;
  private layer2Timer: NodeJS.Timeout | null;
  private layer3Timer: NodeJS.Timeout | null;

  constructor(config?: Partial<HeartbeatConfig>) {
    this.config = {
      layer1Timeout: 1000,
      layer2Timeout: 5000,
      layer3Timeout: 15000,
      retryAttempts: 3,
      retryDelay: 1000,
      ...config
    };
    this.currentMode = 'normal';
    this.signalBuffer = [];
  }

  async fastCheck(): Promise<DetectionResult>;
  async contentCheck(): Promise<DetectionResult>;
  async deepCheck(): Promise<DetectionResult>;
  detect(): Promise<DetectionResult>;
  getCurrentMode(): HeartbeatMode;
  reset(): void;
}
```

## 核心接口说明

### fastCheck() - Layer 1 快速探测

快速探测方法，在页面主线程执行轻量级检查。

**特点：**
- 执行时间：< 50ms
- 检测内容：主线程响应性、基本 DOM 状态
- 适用场景：高频轮询（每秒多次）

**返回值：**
- 当检测到明确信号时，立即返回结果
- 当无法确定时，返回 `mode: 'unknown'`，触发下一层检测

### contentCheck() - Layer 2 内容探测

内容探测方法，通过 CDP 执行脚本检查页面内容状态。

**特点：**
- 执行时间：100ms - 1000ms
- 检测内容：DOM 变化、网络活动、用户交互痕迹
- 适用场景：中等频率检测（每 5-10 秒）

**返回值：**
- 当检测到内容活动信号时，返回对应模式
- 当内容无变化时，可能触发 Layer 3 深度探测

### deepCheck() - Layer 3 深度探测

深度探测方法，执行全面的页面状态分析。

**特点：**
- 执行时间：1s - 15s
- 检测内容：内存状态、进程状态、渲染管线、网络连接
- 适用场景：低频率检测（每 30-60 秒）或异常确认

**返回值：**
- 提供最准确的模式判定
- 包含详细的诊断信息

## 使用示例

```typescript
const detector = new HeartbeatDetector({
  layer1Timeout: 1000,
  layer2Timeout: 5000,
  layer3Timeout: 15000
});

try {
  const result = await detector.detect();
  
  console.log(`检测模式: ${result.mode}`);
  console.log(`置信度: ${result.confidence}`);
  console.log(`判定层级: Layer ${result.layer}`);
  
  switch (result.mode) {
    case 'normal':
      console.log('页面运行正常');
      break;
    case 'frozen':
      console.log('页面已冻结，需要处理');
      break;
    case 'crashed':
      console.log('页面已崩溃，需要恢复');
      break;
  }
} catch (error) {
  console.error('检测失败:', error);
}
```

## 信号收集机制

HeartbeatDetector 维护一个信号缓冲区，用于收集各层检测产生的信号：

```typescript
interface Signal {
  type: SignalType;
  source: 'layer1' | 'layer2' | 'layer3';
  value: any;
  timestamp: number;
  weight: number;  // 信号权重，用于模式判定
}

type SignalType =
  | 'thread_responsive'      // 主线程响应
  | 'thread_blocked'         // 主线程阻塞
  | 'dom_changed'            // DOM 变化
  | 'network_active'         // 网络活动
  | 'user_interaction'       // 用户交互
  | 'memory_pressure'        // 内存压力
  | 'visibility_hidden'      // 页面隐藏
  | 'process_frozen'         // 进程冻结
  | 'render_stopped';        // 渲染停止
```

## 线程安全考虑

HeartbeatDetector 在设计时考虑了多线程/进程环境：

1. **原子操作**：状态变更使用原子操作
2. **锁机制**：信号缓冲区访问使用读写锁
3. **超时保护**：所有检测操作都有超时保护，防止死锁
