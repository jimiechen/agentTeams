# 4.1 HeartbeatDetector 类结构与核心接口

探测器是纯"采集 + 判定"组件，不执行任何恢复动作。

## 类定义

```typescript
class HeartbeatDetector {
  constructor(cdp: CDPClient, workspace: 'PMCLI' | 'DEVCLI');

  // 三层采集接口（独立调用，由 Daemon 按频率编排）
  fastCheck(): Promise<Partial<HeartbeatSignal>>;     // Layer 1 (5s)
  contentCheck(): Promise<Partial<HeartbeatSignal>>;  // Layer 2 (15s)
  deepCheck(): Promise<Partial<HeartbeatSignal>>;     // Layer 3 (60s)

  // 合并三层信号并判定模式
  heartbeat(deep: boolean): Promise<HeartbeatSignal>;

  // 重置内部计数器（任务切换时调用）
  reset(): void;
}
```

## 内部状态

探测器只保留三个必要状态：

| 状态 | 类型 | 用途 |
|------|------|------|
| `lastCharsCheckpoint` | number | 上次文本字符数（用于算 delta） |
| `stalledStartTime` | number | 停滞起始时间戳（用于算 stalledSeconds） |
| `sessionStartTime` | number | session 起始时间（用于算持续时长） |

## 设计原则

### 无状态可重启
不保留历史心跳信号，持久化由 Daemon 负责。探测器本身可以随时重启而不丢失关键信息。

### 职责单一
只做两件事：
1. 采集信号（调用 CDP evaluate）
2. 判定模式（根据信号返回健康模式）

不执行恢复动作，不处理持久化，不发送通知。

## 接口说明

### fastCheck()
- **频率**：5 秒
- **用途**：快速检测显式异常（按钮类）
- **返回**：任务状态 + 按钮存在性

### contentCheck()
- **频率**：15 秒
- **用途**：检测文本增量变化（隐式异常）
- **返回**：字符增量 + 停滞秒数 + 排队状态

### deepCheck()
- **频率**：60 秒
- **用途**：采集深度健康指标
- **返回**：按钮快照 + 资源消耗估算

### heartbeat(deep: boolean)
- **参数**：是否包含深度检查
- **用途**：合并所有信号并判定最终模式
- **返回**：完整的 HeartbeatSignal

### reset()
- **用途**：任务切换时重置内部计数器
- **调用时机**：任务开始、任务完成、任务切换

## 信号结构

```typescript
interface HeartbeatSignal {
  // Layer 1
  taskStatus: 'unknown' | 'completed' | 'in_progress' | 'interrupted' | 'waiting';
  taskStatusText: string;
  hasBackgroundButton: boolean;
  hasCancelButton: boolean;
  hasKeepButton: boolean;
  hasDeleteButton: boolean;
  
  // Layer 2
  lastMessageChars: number;
  charsDelta: number;
  isQueuing: boolean;
  queuePosition: number | null;
  stalledSeconds: number;
  
  // Layer 3
  visibleButtons: Array<{ ariaLabel: string; text: string }>;
  sessionDurationMs: number;
  totalResponseChars: number;
  estimatedTokens: number;
  
  // 派生字段
  mode: HealthMode;
}
```
