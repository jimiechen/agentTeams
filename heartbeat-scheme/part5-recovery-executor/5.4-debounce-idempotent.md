# 5.4 防抖与幂等性保护

## 防抖机制

### 原理
`RecoveryExecutor` 内部维护 `lastRecoveryByMode: Map<HealthMode, number>`，记录每种模式上次恢复的时间戳。

### 实现

```typescript
private lastRecoveryByMode = new Map<HealthMode, number>();
private readonly DEBOUNCE_INTERVAL = 10000; // 10 秒

private canRecover(mode: HealthMode): boolean {
  const lastTime = this.lastRecoveryByMode.get(mode);
  const now = Date.now();
  
  if (lastTime && now - lastTime < this.DEBOUNCE_INTERVAL) {
    console.log(`⏭️ ${mode} 上次恢复在 ${now - lastTime}ms 前，跳过本次`);
    return false;
  }
  
  this.lastRecoveryByMode.set(mode, now);
  return true;
}

async recover(signal: HeartbeatSignal): Promise<RecoveryResult> {
  if (!this.canRecover(signal.mode)) {
    return { attempted: false, action: 'debounced', success: true };
  }
  
  // 执行恢复逻辑...
}
```

### 作用
避免 Layer 1 连续两次检测到同一异常触发双重点击。

## 幂等性保护

### 原理
所有点击操作在 CDP 脚本里先判断按钮存在性，不存在时静默返回 false。

### 实现

```typescript
// CDP 脚本中的幂等保护
const clicked = await this.cdp.evaluate<boolean>(`(() => {
  const buttons = document.querySelectorAll('button');
  for (const btn of buttons) {
    const label = btn.getAttribute('aria-label') || '';
    if (label.includes('后台')) {
      btn.click();
      return true;
    }
  }
  return false; // 按钮不存在，静默返回
})()`);
```

### 作用
按钮消失可能意味着上一次点击已生效，不算错误。

## 状态解耦

### 原理
恢复成功后不立即将模式设为 healthy，由下一次心跳决定。

### 实现

```typescript
// 恢复执行器不修改模式
async recover(signal: HeartbeatSignal): Promise<RecoveryResult> {
  // 执行恢复动作...
  
  // 不修改 signal.mode，由下一次心跳重新判定
  return result;
}
```

### 作用
如果恢复动作真正生效，下一次 Layer 1/2 采集自然不会再看到异常信号，自动回到 healthy。

## 保护机制总结

| 机制 | 目的 | 实现方式 |
|------|------|---------|
| **防抖** | 避免连续触发同一恢复 | 记录上次恢复时间，10秒内跳过 |
| **幂等** | 避免按钮消失时报错 | 按钮不存在时静默返回 |
| **状态解耦** | 避免直接修改模式 | 由下一次心跳判定 |

## 恢复流程中的保护

```
收到 recover 请求
    │
    ▼
防抖检查（10秒内同模式不重复恢复）
    │
    ├─→ 防抖命中 → 返回 { attempted: false }
    │
    └─→ 通过 → 执行恢复动作
            │
            ▼
        幂等检查（按钮是否存在）
            │
            ├─→ 按钮不存在 → 返回 { attempted: true, success: false }（静默）
            │
            └─→ 按钮存在 → 执行点击
                    │
                    ▼
                不修改模式，由下一次心跳判定
```
