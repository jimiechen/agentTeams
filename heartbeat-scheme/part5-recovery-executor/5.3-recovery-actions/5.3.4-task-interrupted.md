# 5.3.4 任务中断：告警挂起

## 实现代码

```typescript
private async notifyAndPause(signal: HeartbeatSignal): Promise<RecoveryResult> {
  await this.notifyAlert('task-interrupted',
    `任务中断，状态："${signal.taskStatusText}"，需人工介入`);
  return { attempted: false, action: 'alert-only', success: true };
}
```

## 设计原则

**明知无法恢复时绝不尝试任何点击**。乱点会让 Trae IDE 进入更糟糕的状态。

## 执行流程

```
收到 task-interrupted 信号
    │
    ▼
发送告警通知到飞书
    │
    ▼
返回 { attempted: false, action: 'alert-only' }
```

## 告警内容

```
【心跳告警】task-interrupted

工作区: PMCLI
时间: 2024-04-29 10:30:00
状态: 中断
原因: "任务执行过程中发生异常"

请人工检查 Trae IDE 状态并决定后续处理。
```

## 后续处理

任务中断后，心跳守护进程会：
1. 停止所有心跳 Timer
2. 记录最终状态到 `heartbeat.jsonl`
3. 等待人工干预或任务重新启动

## 不执行恢复的原因

1. **任务已终止**：任务状态变为"中断"意味着任务已经结束
2. **无法自动恢复**：中断通常需要人工检查根本原因
3. **避免破坏状态**：乱点按钮可能导致数据丢失或状态混乱
