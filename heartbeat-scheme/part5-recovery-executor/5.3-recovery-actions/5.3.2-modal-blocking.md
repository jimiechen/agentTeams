# 5.3.2 弹窗阻塞：按策略点击

## 实现代码

```typescript
private async recoverModalBlocking(signal: HeartbeatSignal): Promise<RecoveryResult> {
  // 安全检查：必须同时存在保留和删除按钮
  if (!signal.hasKeepButton || !signal.hasDeleteButton) {
    return { attempted: true, action: 'modal-blocking', success: false, error: 'invalid state' };
  }

  const target = this.policy.modal_strategy; // 'keep' | 'delete'

  const clicked = await this.cdp.evaluate<boolean>(`(() => {
    const buttons = document.querySelectorAll('button');
    for (const btn of buttons) {
      const label = btn.getAttribute('aria-label') || '';
      const text = btn.textContent?.trim() || '';
      if ('${target}' === 'keep' && (label.includes('保留') || text.includes('保留'))) {
        btn.click();
        return true;
      }
      if ('${target}' === 'delete' && (label.includes('删除') || text.includes('删除'))) {
        btn.click();
        return true;
      }
    }
    return false;
  })()`);

  if (clicked) {
    await this.notifyRecovery('modal-blocking', `点击"${target === 'keep' ? '保留' : '删除'}"`);
    return { attempted: true, action: `click:${target}`, success: true };
  }
  return { attempted: true, action: 'modal-blocking', success: false, error: 'button not found' };
}
```

## 关键安全检查

弹窗出现时必须先确认 `hasKeepButton && hasDeleteButton` 同时为 true（避免单独的"删除"按钮被误点为弹窗）。

## 执行流程

```
收到 modal-blocking 信号
    │
    ▼
安全检查：保留和删除按钮是否都存在
    │
    ├─→ 不满足 → 返回错误
    │
    └─→ 满足 → 获取策略配置（keep / delete）
            │
            ▼
        CDP 执行点击脚本
            │
            ├─→ 按钮存在 → 点击成功
            │       │
            │       ▼
            │  发送恢复通知
            │       │
            │       ▼
            │  返回成功
            │
            └─→ 按钮不存在 → 返回失败
```

## 策略影响

### keep 策略（PMCLI 默认）
- 点击"保留"按钮
- 保留任务历史记录
- 适合文档生成场景

### delete 策略（DEVCLI 默认）
- 点击"删除"按钮
- 删除任务历史记录
- 适合代码修改场景

## 恢复验证

恢复执行后，下一次心跳检查：
- `hasKeepButton` 和 `hasDeleteButton` 是否都为 false
- 任务状态是否为 `completed` 或 `in_progress`
