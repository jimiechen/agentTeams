# 5.3.5 模型排队：被动等待

## 实现代码

```typescript
private async handleQueuing(): Promise<RecoveryResult> {
  // 排队是正常现象，不执行任何恢复动作
  return { attempted: false, action: 'none', success: true };
}
```

## 设计原则

**排队是服务端容量限制导致的正常现象，不应干预**。

## 执行流程

```
收到 model-queuing 信号
    │
    ▼
直接返回 { attempted: false, action: 'none' }
```

## 为什么不干预

1. **服务端限制**：排队是服务端主动限流，客户端无法绕过
2. **等待是唯一选择**：只能等待前面的请求完成
3. **干预可能有害**：频繁重试会加重服务器负担

## 排队期间的行为

### 心跳继续
- Layer 1 继续检测（5秒频率降为15秒）
- Layer 2 继续检测（15秒频率降为30秒）
- Layer 3 继续检测（60秒不变）

### 告警机制
如果排队超过 5 分钟，Layer 3 会触发一次告警通知用户。

### 恢复机制
排队解除后：
- 自动恢复正常心跳频率
- 检测到文本开始增长后转为 healthy

## 排队解除条件

```typescript
// 排队解除的判定条件
if (!signal.isQueuing && signal.charsDelta > 0) {
  // 排队解除，恢复正常
}
```
