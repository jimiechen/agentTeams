# 5.3.1 终端挂起：点击"后台运行"

## 实现代码

```typescript
private async recoverTerminalHang(signal: HeartbeatSignal): Promise<RecoveryResult> {
  const target = this.policy.terminal_strategy; // 'background' | 'cancel'

  const clicked = await this.cdp.evaluate<boolean>(`(() => {
    const buttons = document.querySelectorAll('button');
    for (const btn of buttons) {
      const label = btn.getAttribute('aria-label') || '';
      const text = btn.textContent?.trim() || '';
      if ('${target}' === 'background' && (label.includes('后台') || text.includes('后台运行'))) {
        btn.click();
        return true;
      }
      if ('${target}' === 'cancel' && (label === '取消' || text === '取消')) {
        btn.click();
        return true;
      }
    }
    return false;
  })()`);

  if (clicked) {
    await this.notifyRecovery('terminal-hang', `点击"${target === 'background' ? '后台运行' : '取消'}"`);
    return { attempted: true, action: `click:${target}`, success: true };
  }
  return { attempted: true, action: 'terminal-hang', success: false, error: 'button not found' };
}
```

## 执行流程

```
收到 terminal-hang 信号
    │
    ▼
获取策略配置（background / cancel）
    │
    ▼
CDP 执行点击脚本
    │
    ├─→ 按钮存在 → 点击成功
    │       │
    │       ▼
    │  发送恢复通知
    │       │
    │       ▼
    │  返回 { attempted: true, success: true }
    │
    └─→ 按钮不存在 → 返回 { attempted: true, success: false }
```

## 幂等保护

如果按钮已消失（说明上一次心跳已经点击成功），函数返回 `button not found`，不当成错误处理。

## 策略影响

### background 策略（默认）
- 点击"后台运行"按钮
- 让终端在后台继续执行
- 任务可以继续进行

### cancel 策略
- 点击"取消"按钮
- 终止当前终端执行
- 可能导致任务失败

## 恢复验证

恢复执行后，下一次心跳检查：
- `hasBackgroundButton` 是否为 false
- 任务状态是否为 `in_progress`
