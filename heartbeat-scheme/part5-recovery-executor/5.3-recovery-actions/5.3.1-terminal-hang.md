# 5.3.1 Terminal Hang 恢复动作

## 5.3.1.1 问题定义

**terminal-hang** 是指终端命令执行过程中出现的"后台运行"按钮状态。这表示命令正在执行但可能耗时较长，需要用户决定是否让其在后台运行。

### 识别特征

| 特征 | 说明 |
|------|------|
| **触发条件** | 检测到"后台运行"按钮 |
| **检测层级** | Layer 1（快速心跳） |
| **时效要求** | 5 秒内识别 |
| **自治可行性** | ✅ 可自动点击按钮 |

### 状态判定

```typescript
interface TerminalHangSignal {
  hasBackgroundBtn: boolean;
  hasCancelBtn: boolean;
  taskStatus: string;
  terminalOutput: string;
}

function isTerminalHang(signal: TerminalHangSignal): boolean {
  return signal.hasBackgroundBtn && 
         signal.taskStatus === 'running' &&
         !signal.hasCancelBtn;
}
```

## 5.3.1.4 恢复执行器实现

```typescript
class TerminalHangRecoveryExecutor implements ActionExecutor {
  readonly type = 'terminal-hang';
  readonly action = 'click-background';
  
  private page: Page;
  private buttonFinder: ButtonFinder;
  private permissionChecker: PermissionChecker;
  
  constructor(page: Page) {
    this.page = page;
    this.buttonFinder = new ButtonFinder(page);
    this.permissionChecker = new PermissionChecker();
  }

  async execute(context: RecoveryContext): Promise<RecoveryResult> {
    const startTime = Date.now();
    
    console.log(`[TerminalHang] 开始恢复工作区 ${context.workspaceId}`);
    
    const permissionResult = this.permissionChecker.checkPermission('background');
    if (!permissionResult.allowed) {
      return {
        success: false,
        action: this.action,
        reason: permissionResult.reason,
        durationMs: Date.now() - startTime,
        nextState: 'terminal-hang'
      };
    }
    
    const buttonResult = await this.buttonFinder.findButton('background');
    
    if (!buttonResult.found || !buttonResult.element) {
      return {
        success: false,
        action: this.action,
        reason: '未找到"后台运行"按钮',
        durationMs: Date.now() - startTime,
        nextState: 'terminal-hang'
      };
    }
    
    try {
      await buttonResult.element.click();
      
      this.permissionChecker.recordClick('background');
      
      console.log(`[TerminalHang] 已点击"后台运行"按钮`);
      
      return {
        success: true,
        action: this.action,
        reason: '成功点击"后台运行"按钮',
        durationMs: Date.now() - startTime,
        nextState: 'verifying'
      };
    } catch (error) {
      return {
        success: false,
        action: this.action,
        reason: `点击失败: ${error instanceof Error ? error.message : '未知错误'}`,
        durationMs: Date.now() - startTime,
        nextState: 'terminal-hang'
      };
    }
  }

  async canExecute(context: RecoveryContext): Promise<boolean> {
    const buttonResult = await this.buttonFinder.findButton('background');
    return buttonResult.found;
  }
}
```

## 5.3.1.5 PMCLI/DEVCLI 策略差异

### PMCLI 策略

```typescript
const PMCLI_TERMINAL_HANG_CONFIG = {
  autoExecute: true,
  requiresConfirmation: false,
  maxRetries: 3,
  verifyingWindowMs: 30000,
  successThreshold: 0,
  postAction: 'log-only'
};
```

**PMCLI 特点**：
- 自动点击"后台运行"按钮
- 不需要用户确认
- 验证窗口 30 秒
- 仅记录日志，不发送告警

### DEVCLI 策略

```typescript
const DEVCLI_TERMINAL_HANG_CONFIG = {
  autoExecute: false,
  requiresConfirmation: true,
  maxRetries: 3,
  verifyingWindowMs: 30000,
  successThreshold: 0,
  postAction: 'notify-user',
  preserveContext: true
};
```

**DEVCLI 特点**：
- 需要用户确认后才执行
- 保留终端上下文
- 通知用户恢复动作
- 验证窗口 30 秒

## 5.3.1.6 验证逻辑

```typescript
class TerminalHangVerifier {
  private readonly VERIFY_WINDOW_MS = 30000;
  private readonly CHECK_INTERVAL_MS = 5000;
  
  async verify(page: Page): Promise<boolean> {
    const startTime = Date.now();
    
    while (Date.now() - startTime < this.VERIFY_WINDOW_MS) {
      const buttonResult = await this.findBackgroundButton(page);
      
      if (!buttonResult.found) {
        console.log('[TerminalHang] 按钮已消失，恢复成功');
        return true;
      }
      
      const taskStatus = await this.getTaskStatus(page);
      
      if (taskStatus === 'completed' || taskStatus === 'running') {
        console.log('[TerminalHang] 任务状态正常，恢复成功');
        return true;
      }
      
      await this.sleep(this.CHECK_INTERVAL_MS);
    }
    
    console.log('[TerminalHang] 验证窗口超时');
    return false;
  }
  
  private async findBackgroundButton(page: Page): Promise<{
    found: boolean;
    element: Locator | null;
  }> {
    for (const selector of TERMINAL_HANG_SELECTORS.backgroundBtn) {
      try {
        const element = page.locator(selector).first();
        const isVisible = await element.isVisible({ timeout: 1000 });
        
        if (isVisible) {
          return { found: true, element };
        }
      } catch {
        continue;
      }
    }
    
    return { found: false, element: null };
  }
  
  private async getTaskStatus(page: Page): Promise<string> {
    try {
      const statusElement = page.locator('[data-testid="task-status"]').first();
      return await statusElement.textContent() || 'unknown';
    } catch {
      return 'unknown';
    }
  }
  
  private sleep(ms: number): Promise<void> {
    return new Promise(resolve => setTimeout(resolve, ms));
  }
}
```

## 5.3.1.7 日志与监控

```typescript
interface TerminalHangLog {
  timestamp: string;
  workspaceId: string;
  taskId: string;
  action: 'click-background';
  result: 'success' | 'failed' | 'skipped';
  reason: string;
  durationMs: number;
  terminalSnapshot?: string;
}

class TerminalHangLogger {
  private logs: TerminalHangLog[] = [];
  
  log(entry: Omit<TerminalHangLog, 'timestamp'>): void {
    const log: TerminalHangLog = {
      ...entry,
      timestamp: new Date().toISOString()
    };
    
    this.logs.push(log);
    this.writeToFile(log);
  }
  
  private writeToFile(log: TerminalHangLog): void {
    const logPath = `logs/terminal-hang-${new Date().toISOString().split('T')[0]}.jsonl`;
    fs.appendFileSync(logPath, JSON.stringify(log) + '\n');
  }
  
  getStats(): {
    total: number;
    success: number;
    failed: number;
    avgDurationMs: number;
  } {
    const total = this.logs.length;
    const success = this.logs.filter(l => l.result === 'success').length;
    const failed = this.logs.filter(l => l.result === 'failed').length;
    
    const totalDuration = this.logs.reduce((sum, l) => sum + l.durationMs, 0);
    const avgDurationMs = total > 0 ? totalDuration / total : 0;
    
    return { total, success, failed, avgDurationMs };
  }
}
```

## 5.3.1.8 边界情况处理

### 情况一：按钮已消失

```typescript
async function handleButtonGone(page: Page): Promise<RecoveryResult> {
  console.log('[TerminalHang] 按钮已不存在，可能已被其他方式处理');
  
  return {
    success: true,
    action: 'none',
    reason: '按钮已消失，无需处理',
    durationMs: 0,
    nextState: 'healthy'
  };
}
```

### 情况二：点击后按钮仍存在

```typescript
async function handleButtonStillPresent(page: Page): Promise<RecoveryResult> {
  console.log('[TerminalHang] 点击后按钮仍存在，尝试再次点击');
  
  const retryResult = await this.retryClick(page);
  
  if (!retryResult.success) {
    return {
      success: false,
      action: 'click-background',
      reason: '多次点击后按钮仍存在',
      durationMs: retryResult.durationMs,
      nextState: 'terminal-hang'
    };
  }
  
  return retryResult;
}
```

### 情况三：同时出现多个按钮

```typescript
async function handleMultipleButtons(page: Page): Promise<RecoveryResult> {
  console.log('[TerminalHang] 检测到多个按钮，按优先级处理');
  
  const priority = ['background', 'cancel'];
  
  for (const buttonType of priority) {
    const result = await this.buttonFinder.findButton(buttonType);
    
    if (result.found && result.element) {
      await result.element.click();
      
      return {
        success: true,
        action: `click-${buttonType}`,
        reason: `点击了 ${buttonType} 按钮`,
        durationMs: 0,
        nextState: 'verifying'
      };
    }
  }
  
  return {
    success: false,
    action: 'none',
    reason: '无法处理多个按钮',
    durationMs: 0,
    nextState: 'terminal-hang'
  };
}
```
