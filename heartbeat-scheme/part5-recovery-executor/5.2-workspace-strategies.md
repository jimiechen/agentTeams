# 5.2 工作区策略配置

## 策略差异

PMCLI 和 DEVCLI 对"破坏性操作"的容忍度不同，策略通过配置区分。

## 配置格式

```yaml
# config/heartbeat.yaml
policies:
  PMCLI:
    modal_strategy: keep         # 弹窗默认保留文件
    terminal_strategy: background # 终端默认后台运行
    stalled_retry_limit: 2       # 停滞最多重试 2 次
    alert_on_first_stall: false  # 第一次停滞不告警

  DEVCLI:
    modal_strategy: delete       # 允许覆盖文件
    terminal_strategy: background
    stalled_retry_limit: 3       # 更激进
    alert_on_first_stall: false
```

## 策略说明

| 策略项 | PMCLI | DEVCLI | 说明 |
|-------|-------|--------|------|
| `modal_strategy` | keep | delete | 弹窗处理策略 |
| `terminal_strategy` | background | background | 终端处理策略 |
| `stalled_retry_limit` | 2 | 3 | 停滞重试上限 |
| `alert_on_first_stall` | false | false | 首次停滞是否告警 |

## 策略意义

### PMCLI（文档生成）
- 文件历史价值高，默认"保留"
- 重试次数保守（避免文档重复生成）

### DEVCLI（代码修改）
- 需要允许覆盖，默认"删除"
- 重试次数更激进（代码修改可接受多次尝试）

## 策略决策链

```
用户指令
    │
    ▼
路由到对应工作区
    │
    ▼
加载工作区策略配置
    │
    ▼
策略影响恢复动作
    │
    ├─→ modal-blocking → 按 modal_strategy 点击按钮
    ├─→ terminal-hang → 按 terminal_strategy 点击按钮
    └─→ model-stalled → 按 stalled_retry_limit 重试
```

## 策略热更新

策略配置支持热更新，无需重启服务：

```typescript
async reloadPolicy(workspace: string): Promise<void> {
  const config = await loadConfig('config/heartbeat.yaml');
  this.policy = config.policies[workspace];
}
```

## 策略校验

### 启动时校验
- 检查所有工作区是否有配置
- 检查策略值是否在允许范围内

### 运行时校验
- 每次恢复前检查策略是否有效
- 无效策略使用默认值

## 默认策略

```yaml
default:
  modal_strategy: keep
  terminal_strategy: background
  stalled_retry_limit: 2
  alert_on_first_stall: false
```
