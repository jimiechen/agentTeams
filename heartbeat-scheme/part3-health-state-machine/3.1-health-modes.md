# 3.1 7种健康模式的定义与互斥关系

## 模式定义

### healthy
- **含义**：任务正常执行中，无需干预
- **特征**：任务状态为"进行中"，无异常按钮，文本正常增长
- **处理**：持续监控，无需动作

### completed
- **含义**：任务已完成，停止心跳
- **特征**：任务状态变为"完成"
- **处理**：停止所有心跳层，清理资源

### model-stalled
- **含义**：模型停滞，可自动恢复
- **特征**：连续 30 秒文本无增长，且无排队状态
- **处理**：执行 stop + retry，最多重试 3 次

### terminal-hang
- **含义**：终端挂起，可自动恢复
- **特征**：出现"后台运行"按钮
- **处理**：点击"后台运行"按钮，最多重试 2 次

### modal-blocking
- **含义**：弹窗阻塞，可按策略自动恢复
- **特征**：出现"保留/删除"按钮
- **处理**：按预设策略点击按钮，最多重试 1 次

### model-queuing
- **含义**：模型排队，被动等待
- **特征**：消息中出现"排在第 X 位"文本
- **处理**：不干预，被动等待

### task-interrupted
- **含义**：任务中断，需人工介入
- **特征**：任务状态变为"中断"
- **处理**：发送告警，停止心跳

### unknown
- **含义**：信号不完整，跳过本轮判定
- **特征**：关键信号缺失或过期
- **处理**：跳过本轮，等待下次采样

## 模式互斥关系

任一时刻只能处于一种模式。判定函数必须保证返回单一结果。

### 互斥规则
1. **优先级最高**：task-interrupted（终态）
2. **显式异常优先**：terminal-hang、modal-blocking
3. **非异常次之**：model-queuing
4. **隐式异常最后**：model-stalled
5. **正常状态**：healthy、completed
6. **兜底**：unknown

## 模式转换约束

| 模式 | 可转入 | 不可转入 |
|------|-------|---------|
| healthy | 所有异常模式 + completed | - |
| model-stalled | healthy、task-interrupted | completed（未完成） |
| terminal-hang | healthy | completed、task-interrupted |
| modal-blocking | healthy | completed、task-interrupted |
| model-queuing | healthy | task-interrupted |
| task-interrupted | -（终态） | 任何模式 |
| completed | -（终态） | 任何模式 |

## 模式优先级总结

```
优先级从高到低：
1. task-interrupted   （终态优先）
2. modal-blocking     （弹窗优先处理）
3. terminal-hang      （终端优先）
4. model-queuing      （排队非异常）
5. model-stalled      （停滞最后判定）
6. completed          （正常完成）
7. healthy            （正常执行）
8. unknown            （信号缺失）
```
