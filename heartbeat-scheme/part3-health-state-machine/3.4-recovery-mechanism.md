# 3.4 状态回落机制

## 3.4.1 回落机制概述

状态回落机制处理**恢复动作后的状态验证**。当执行恢复动作（如 stop+retry）后，需要验证是否真正恢复成功。

核心概念：
- **verifying 窗口**：恢复动作后的验证时间窗口
- **重试计数**：允许的最大恢复尝试次数
- **回落判定**：验证窗口内判断恢复是否成功

## 3.4.2 verifying 验证窗口

### 3.4.2.1 窗口定义

```typescript
interface VerifyingWindow {
  durationMs: number;        // 窗口时长，默认 60_000（60秒）
  checkIntervalMs: number;   // 检查间隔，默认 5_000（5秒）
  successThreshold: number;  // 成功阈值：文本增长字数
}
```

### 3.4.2.2 窗口启动条件

当执行以下恢复动作后，启动 verifying 窗口：

| 恢复动作 | 触发模式 | 窗口时长 |
|---------|---------|---------|
| stop + retry | `model-stalled` | 60 秒 |
| 点击"后台运行" | `terminal-hang` | 30 秒 |
| 处理模态弹窗 | `modal-blocking` | 30 秒 |

### 3.4.2.3 窗口内检测逻辑

```typescript
async function verifyRecovery(window: VerifyingWindow): Promise<boolean> {
  const startTime = Date.now();
  let lastCharCount = await getCharCount();
  
  while (Date.now() - startTime < window.durationMs) {
    await sleep(window.checkIntervalMs);
    
    const currentCharCount = await getCharCount();
    const charsDelta = currentCharCount - lastCharCount;
    
    if (charsDelta >= window.successThreshold) {
      return true;  // 恢复成功
    }
    
    lastCharCount = currentCharCount;
  }
  
  return false;  // 窗口结束仍未恢复
}
```

### 3.4.2.4 窗口状态转移

```
┌──────────────┐     执行恢复动作     ┌──────────────┐
│ model-stalled│──────────────────► │  verifying   │
└──────────────┘                     └──────┬───────┘
                                            │
                              ┌─────────────┼─────────────┐
                              │             │             │
                              ▼             ▼             ▼
                        ┌──────────┐ ┌──────────┐ ┌──────────────┐
                        │ healthy  │ │ 重试再次 │ │ interrupted  │
                        │(恢复成功)│ │ verifying│ │  (重试耗尽)  │
                        └──────────┘ └──────────┘ └──────────────┘
```

## 3.4.3 重试计数策略

### 3.4.3.1 重试配置

```typescript
interface RetryConfig {
  maxAttempts: number;       // 最大重试次数，默认 3
  backoffMs: number;         // 重试退避时间，默认 10_000（10秒）
  resetAfterMs: number;      // 成功后重置计数器的时间，默认 300_000（5分钟）
}
```

### 3.4.3.2 重试计数器

每个工作区维护独立的重试计数器：

```typescript
interface RetryCounter {
  workspaceId: string;
  currentAttempts: number;
  lastAttemptTime: string;
  lastSuccessTime: string;
}
```

### 3.4.3.3 重试流程

```
第1次停滞 ──► stop+retry ──► verifying ──► 失败 ──► attempts = 1
                                                      │
第2次停滞 ──► stop+retry ──► verifying ──► 失败 ──► attempts = 2
                                                      │
第3次停滞 ──► stop+retry ──► verifying ──► 失败 ──► attempts = 3
                                                      │
                                                      ▼
                                              进入 interrupted 终态
                                              发送 critical 告警
```

### 3.4.3.4 重试计数器重置

计数器在以下情况重置：

1. **恢复成功**：verifying 窗口内检测到文本增长
2. **任务完成**：任务进入 `completed` 状态
3. **超时重置**：距离上次成功超过 `resetAfterMs`（5分钟）

```typescript
function shouldResetCounter(counter: RetryCounter): boolean {
  if (counter.currentAttempts === 0) return false;
  
  const timeSinceSuccess = Date.now() - new Date(counter.lastSuccessTime).getTime();
  return timeSinceSuccess > RETRY_CONFIG.resetAfterMs;
}
```

## 3.4.4 回落判定规则

### 3.4.4.1 成功判定

verifying 窗口内满足以下条件判定为恢复成功：

| 条件 | 阈值 | 说明 |
|------|------|------|
| 文本增长 | ≥ 10 字符 | AI 开始响应 |
| 无新异常信号 | - | 无新弹窗/按钮 |
| 任务状态正常 | `running` | 任务未被中断 |

### 3.4.4.2 失败判定

verifying 窗口内出现以下情况判定为恢复失败：

| 条件 | 说明 |
|------|------|
| 窗口结束时文本无增长 | AI 仍未响应 |
| 检测到新的异常模式 | 出现新弹窗/中断 |
| stop 按钮不存在 | 无法再次重试 |

### 3.4.4.3 边界情况处理

**情况一：窗口内出现新的异常**

```
verifying ──► 检测到 modal-blocking ──► 暂停 verifying
                                              │
                    处理弹窗 ──► 继续 verifying ◄─┘
```

**情况二：窗口内任务完成**

```
verifying ──► 检测到 completed ──► 直接进入 completed 终态
```

**情况三：窗口内任务中断**

```
verifying ──► 检测到 interrupted ──► 直接进入 interrupted 终态
```

## 3.4.6 回落配置参数

所有回落参数支持通过 Admin API 动态调整：

```typescript
interface FallbackConfig {
  verifying: {
    stalledWindowMs: number;      // model-stalled 验证窗口，默认 60_000
    terminalWindowMs: number;     // terminal-hang 验证窗口，默认 30_000
    modalWindowMs: number;        // modal-blocking 验证窗口，默认 30_000
    checkIntervalMs: number;      // 检查间隔，默认 5_000
    successThreshold: number;     // 成功阈值（字符数），默认 10
  };
  retry: {
    maxAttempts: number;          // 最大重试次数，默认 3
    backoffMs: number;            // 重试退避时间，默认 10_000
    resetAfterMs: number;         // 计数器重置时间，默认 300_000
  };
}
```

## 3.4.7 回落日志与监控

### 3.4.7.1 日志记录

每次回落过程记录详细日志：

```typescript
interface FallbackLog {
  timestamp: string;
  workspaceId: string;
  taskId: string;
  triggerMode: HealthMode;
  action: string;
  verifyingWindow: {
    startTime: string;
    endTime: string;
    checks: Array<{
      time: string;
      charCount: number;
      charsDelta: number;
    }>;
  };
  result: 'success' | 'retry' | 'exhausted';
  attemptNumber: number;
  totalDurationMs: number;
}
```

### 3.4.7.2 监控指标

```typescript
interface FallbackMetrics {
  totalRecoveries: number;
  successfulRecoveries: number;
  failedRecoveries: number;
  averageRecoveryTimeMs: number;
  retryDistribution: {
    firstAttempt: number;
    secondAttempt: number;
    thirdAttempt: number;
    exhausted: number;
  };
  modeDistribution: {
    modelStalled: number;
    terminalHang: number;
    modalBlocking: number;
  };
}
```

日志写入 `logs/heartbeat-fallback-{date}.jsonl`，指标上报到监控系统。
