# 3.2 状态转移矩阵

## 3.2.1 转移矩阵概览

状态转移矩阵定义了 7 种模式之间的合法转换路径：

```
                 ┌─────────────────────────────────────────────────┐
                 │                                                 │
                 ▼                                                 │
            ┌─────────┐                                            │
            │ healthy │◄───────────────────────────────────────┐   │
            └────┬────┘                                        │   │
                 │                                             │   │
        ┌────────┼────────┬────────────┬────────────┐         │   │
        ▼        ▼        ▼            ▼            ▼         │   │
   ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐│   │
   │completed│ │ stalled │ │terminal │ │ modal   │ │ queuing ││   │
   │  (终态) │ │         │ │  hang   │ │blocking │ │         ││   │
   └─────────┘ └────┬────┘ └────┬────┘ └────┬────┘ └────┬────┘│   │
                    │           │           │            │     │   │
                    │           │           │            │     │   │
                    ▼           │           │            │     │   │
               ┌─────────┐      │           │            │     │   │
               │verifying│──────┴───────────┴────────────┘     │   │
               └────┬────┘                                      │   │
                    │                                           │   │
            ┌───────┴───────┐                                   │   │
            ▼               ▼                                   │   │
       ┌─────────┐   ┌──────────────┐                          │   │
       │ healthy │   │ interrupted  │◄─────────────────────────┘   │
       │(恢复成功)│   │   (终态)     │                              │
       └─────────┘   └──────────────┘                              │
                                                                    │
```

## 3.2.2 完整转移矩阵表

| 当前状态 | 目标状态 | 触发条件 | 动作 |
|---------|---------|---------|------|
| `healthy` | `completed` | 任务状态 = completed | 停止心跳，生成摘要 |
| `healthy` | `model-stalled` | 文本 30 秒无增长 | 进入恢复流程 |
| `healthy` | `terminal-hang` | 检测到"后台运行"按钮 | 点击"后台运行" |
| `healthy` | `modal-blocking` | 检测到模态弹窗 | 自动处理弹窗 |
| `healthy` | `model-queuing` | 检测到排队提示 | 延长超时，等待 |
| `healthy` | `task-interrupted` | 任务状态 = 中断 | 发送 critical 告警 |
| `model-stalled` | `verifying` | 执行 stop+retry | 启动验证窗口 |
| `model-stalled` | `task-interrupted` | 重试次数耗尽 | 发送 critical 告警 |
| `terminal-hang` | `healthy` | 点击"后台运行"成功 | 发送 info 通知 |
| `terminal-hang` | `task-interrupted` | 用户点击"取消" | 发送 warning 通知 |
| `modal-blocking` | `healthy` | 处理弹窗成功 | 发送 info 通知 |
| `modal-blocking` | `task-interrupted` | 弹窗无法处理 | 发送 warning 通知 |
| `model-queuing` | `healthy` | 排队结束，AI 响应 | 无 |
| `model-queuing` | `model-stalled` | 排队提示消失，仍无响应 | 进入恢复流程 |
| `verifying` | `healthy` | 验证窗口内文本增长 | 发送 warning 通知 |
| `verifying` | `model-stalled` | 验证失败，重试次数未耗尽 | 再次 stop+retry |
| `verifying` | `task-interrupted` | 验证失败，重试次数耗尽 | 发送 critical 告警 |
| `unknown` | `healthy` | 检测到明确正常信号 | 无 |
| `unknown` | 其他模式 | 检测到明确异常信号 | 按对应模式处理 |
| `unknown` | `task-interrupted` | 超过 5 分钟无法判断 | 发送 critical 告警 |

## 3.2.3 状态转移触发条件详解

### 3.2.3.1 healthy → model-stalled

**触发条件**：
```typescript
const conditions = {
  charsDelta: 0,                    // 文本无增长
  consecutiveCount: 2,              // 连续 2 次检测
  intervalMs: 15_000,               // 检测间隔 15 秒
  totalDurationMs: 30_000,          // 累计 30 秒
  excludePatterns: ['排在第', '排队'] // 排除排队情况
};
```

**判定逻辑**：
```
IF charsDelta == 0 
   AND consecutiveCount >= 2
   AND NOT containsQueueHint()
THEN mode = 'model-stalled'
```

### 3.2.3.2 healthy → terminal-hang

**触发条件**：
```typescript
const conditions = {
  domSelector: '[class*="background-run"]',  // 后台运行按钮
  buttonText: ['后台运行', 'background'],
  visibility: true
};
```

**判定逻辑**：
```
IF elementExists('[class*="background-run"]')
   AND isVisible(button)
   AND buttonText matches '后台运行'
THEN mode = 'terminal-hang'
```

### 3.2.3.3 healthy → modal-blocking

**触发条件**：
```typescript
const conditions = {
  domSelectors: [
    '[class*="modal"]',
    '[role="dialog"]',
    '.ant-modal',
    '[class*="confirm"]'
  ],
  hasButtons: true,
  isBlocking: true
};
```

**判定逻辑**：
```
IF elementExists(modalSelectors)
   AND hasButtons(modal)
   AND isBlocking(modal)
THEN mode = 'modal-blocking'
```

### 3.2.3.4 healthy → model-queuing

**触发条件**：
```typescript
const conditions = {
  textPatterns: ['排在第', '排队', '请稍候', '等待中'],
  location: 'lastMessage',
  charsDelta: 0
};
```

**判定逻辑**：
```
IF lastMessage contains '排在第'
   AND charsDelta == 0
THEN mode = 'model-queuing'
```

### 3.2.3.5 healthy → task-interrupted

**触发条件**：
```typescript
const conditions = {
  taskStatus: ['中断', 'interrupted', 'cancelled'],
  location: 'taskList'
};
```

**判定逻辑**：
```
IF taskItem.textContent contains '中断'
THEN mode = 'task-interrupted'
```

## 3.2.4 终态定义

以下状态为终态，一旦进入不再转移：

| 终态 | 进入条件 | 后续处理 |
|------|---------|---------|
| `completed` | 任务成功完成 | 生成心跳摘要，写入 runs/*.md |
| `task-interrupted` | 任务被中断或恢复失败 | 发送告警，等待人工介入 |

## 3.2.6 状态转移计数器

每次状态转移都更新计数器，用于监控和告警：

```typescript
interface TransitionCounters {
  healthy_to_stalled: number;
  healthy_to_terminal_hang: number;
  healthy_to_modal_blocking: number;
  healthy_to_queuing: number;
  stalled_recovery_success: number;
  stalled_recovery_failed: number;
  terminal_hang_resolved: number;
  modal_blocking_resolved: number;
  total_recoveries: number;
}
```

计数器写入 `runs/{taskId}.jsonl` 和心跳摘要。
