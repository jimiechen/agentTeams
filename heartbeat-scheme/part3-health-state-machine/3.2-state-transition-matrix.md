# 3.2 状态转移矩阵

## 转移矩阵表

| 当前模式 | 可转入模式 | 触发条件 |
|---------|-----------|---------|
| healthy | model-stalled | 连续 30s 无文本增长且无排队 |
| healthy | terminal-hang | 出现"后台运行"按钮 |
| healthy | modal-blocking | 出现"保留/删除"按钮 |
| healthy | model-queuing | 文本出现"排在第 X 位" |
| healthy | task-interrupted | 任务状态变为"中断" |
| healthy | completed | 任务状态变为"完成" |
| model-stalled | healthy | 恢复成功后文本重新增长 |
| model-stalled | task-interrupted | 重试次数耗尽（≥3次） |
| terminal-hang | healthy | 点击"后台运行"后按钮消失 |
| terminal-hang | task-interrupted | 重试次数耗尽（≥2次） |
| modal-blocking | healthy | 点击按钮后弹窗消失 |
| modal-blocking | task-interrupted | 重试次数耗尽（≥1次） |
| model-queuing | healthy | 排队文本消失且开始流式输出 |
| model-queuing | task-interrupted | 任务状态变为"中断" |
| model-queuing | completed | 任务状态变为"完成" |
| 任意 | completed | 任务状态变为"完成" |
| task-interrupted | 终态 | 不再转移 |
| completed | 终态 | 不再转移 |

## 转移条件详解

### healthy → model-stalled
```javascript
// 连续 30 秒无文本增长且无排队
if (lastSignal.layer2.stalledSeconds >= 30 && !lastSignal.layer2.isQueuing) {
    return 'model-stalled';
}
```

### healthy → terminal-hang
```javascript
// 出现"后台运行"按钮
if (lastSignal.layer1.hasBackgroundButton) {
    return 'terminal-hang';
}
```

### healthy → modal-blocking
```javascript
// 出现"保留"或"删除"按钮
if (lastSignal.layer1.hasKeepButton || lastSignal.layer1.hasDeleteButton) {
    return 'modal-blocking';
}
```

### healthy → model-queuing
```javascript
// 文本出现排队标识
if (lastSignal.layer2.isQueuing) {
    return 'model-queuing';
}
```

### model-stalled → task-interrupted
```javascript
// 重试次数耗尽
if (lastSignal.recoveryAttempts.modelStalled >= 3) {
    return 'task-interrupted';
}
```

### model-queuing → healthy
```javascript
// 排队解除且开始流式输出
if (!lastSignal.layer2.isQueuing && lastSignal.layer2.charsDelta > 0) {
    return 'healthy';
}
```

## 转移优先级规则

当多个转移条件同时满足时，按以下优先级选择：

1. **终态优先**：completed 和 task-interrupted 优先于其他转移
2. **显式异常优先**：terminal-hang、modal-blocking 优先于 model-stalled
3. **排队优先**：model-queuing 优先于 model-stalled（避免误判）
4. **正常状态最后**：healthy 只在无其他条件满足时返回

## 转移代码实现

```javascript
function determineNextMode(currentMode) {
    const { taskStatus, hasBackgroundButton, hasKeepButton, hasDeleteButton } = lastSignal.layer1;
    const { isQueuing, stalledSeconds, charsDelta } = lastSignal.layer2;
    const { modelStalled, terminalHang, modalBlocking } = lastSignal.recoveryAttempts;
    
    // 终态检查
    if (taskStatus === 'completed') return 'completed';
    if (taskStatus === 'interrupted') return 'task-interrupted';
    
    // 显式异常检查
    if (hasKeepButton || hasDeleteButton) return 'modal-blocking';
    if (hasBackgroundButton) return 'terminal-hang';
    
    // 排队检查
    if (isQueuing) return 'model-queuing';
    
    // 停滞检查（需要看当前模式和重试次数）
    if (stalledSeconds >= 30 && modelStalled < 3) return 'model-stalled';
    
    // 恢复成功检查
    if (currentMode !== 'healthy' && charsDelta > 0) {
        // 重置重试计数器
        lastSignal.recoveryAttempts = { modelStalled: 0, terminalHang: 0, modalBlocking: 0 };
        return 'healthy';
    }
    
    // 重试耗尽检查
    if (currentMode === 'model-stalled' && modelStalled >= 3) return 'task-interrupted';
    if (currentMode === 'terminal-hang' && terminalHang >= 2) return 'task-interrupted';
    if (currentMode === 'modal-blocking' && modalBlocking >= 1) return 'task-interrupted';
    
    return 'healthy';
}
```

## 转移约束说明

### 不可转移场景
- **task-interrupted**：无法转移到任何其他模式（终态）
- **completed**：无法转移到任何其他模式（终态）
- **model-stalled**：无法直接转移到 completed（必须先恢复到 healthy）

### 特殊转移场景
- **任意模式 → completed**：任务完成是最高优先级，任何模式下检测到完成都立即转移
- **model-queuing → task-interrupted**：排队中也可能出现任务中断
