# 6.4 异常退出与资源清理

## 异常情况分类

### 情况一：CDP 连接断开

**处理策略**：

```typescript
// 在 Timer 中捕获 CDP 错误
this.timers.fast = setInterval(async () => {
  try {
    const signal = await this.detector.heartbeat(false);
    // ...
  } catch (err) {
    this.logError('layer1', err);
    this.cdpErrorCount++;
    
    // 连续 5 次失败自动停止
    if (this.cdpErrorCount >= 5) {
      console.log('❌ 连续 5 次 CDP 失败，停止心跳');
      await this.notifyAlert('critical', 'CDP 连接异常，心跳已停止');
      this.stop();
    }
  }
}, 5000);
```

**恢复机制**：
- 记录错误到 `heartbeat-errors.log`
- 连续 5 次失败后自动停止
- 发送告警通知

### 情况二：Node 进程退出信号

**处理策略**：

```typescript
// 在 runner-multi.ts 中处理退出信号
process.on('SIGINT', async () => {
  console.log('📤 收到 SIGINT，正在停止心跳守护进程...');
  for (const daemon of activeDaemons) {
    daemon.stop();
  }
  process.exit(0);
});

process.on('SIGTERM', async () => {
  console.log('📤 收到 SIGTERM，正在停止心跳守护进程...');
  for (const daemon of activeDaemons) {
    daemon.stop();
  }
  process.exit(0);
});
```

**关键设计**：遍历所有活跃的守护进程并调用 `stop()`，确保所有 Timer 被清理。

### 情况三：Trae IDE 崩溃

**处理策略**：

```typescript
// CDP evaluate 失败时的处理
try {
  const result = await this.cdp.evaluate(SCRIPT);
} catch (err) {
  if (err.message.includes('target closed')) {
    console.log('❌ Trae IDE 可能已关闭');
    await this.notifyAlert('critical', 'Trae IDE 连接断开，请检查 IDE 状态');
    this.stop();
  }
}
```

**恢复机制**：
- 检测 `target closed` 错误
- 发送告警通知用户
- 停止心跳守护进程

## 资源清理清单

### 必须清理的资源

| 资源 | 清理方式 |
|------|---------|
| **Timer** | `clearInterval()` |
| **CDP 连接** | 由 CDPClient 自动管理 |
| **日志文件句柄** | Node.js 自动关闭 |
| **内存状态** | 变量置空 |

### 清理代码

```typescript
stop(): void {
  this.running = false;
  
  // 停止所有 Timer
  if (this.timers.fast) clearInterval(this.timers.fast);
  if (this.timers.content) clearInterval(this.timers.content);
  if (this.timers.deep) clearInterval(this.timers.deep);
  
  // 清理引用
  this.timers = {};
  
  // 重置状态
  this.cdpErrorCount = 0;
  
  // 调用停止钩子
  this.onStop();
}
```

## 错误日志

### 日志格式

```typescript
private logError(layer: string, err: Error): void {
  const errorEntry = {
    ts: Date.now(),
    workspace: this.workspace,
    layer,
    error: {
      message: err.message,
      stack: err.stack?.slice(0, 200),
    },
  };
  appendFileSync(this.errorLogPath, JSON.stringify(errorEntry) + '\n');
}
```

### 日志路径

`workspaces/{workspace}/runs/heartbeat-errors.log`

## 总结

### 异常处理策略

| 异常类型 | 检测方式 | 处理动作 |
|---------|---------|---------|
| CDP 连接断开 | 捕获 evaluate 错误 | 计数，连续 5 次失败停止 |
| 进程退出信号 | 监听 SIGINT/SIGTERM | 遍历停止所有守护进程 |
| Trae IDE 崩溃 | 检测 target closed 错误 | 发送告警，停止心跳 |

### 资源清理原则
- 主动清理所有 Timer
- 依赖 GC 清理内存
- 确保没有资源泄漏
