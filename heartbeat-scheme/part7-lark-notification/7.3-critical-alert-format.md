# 7.3 critical 级告警格式

## 7.3.1 设计原则

critical 级告警代表严重问题，设计遵循以下原则：

1. **即时性**：立即发送，无延迟
2. **强提醒**：@相关用户，触发推送通知
3. **可操作**：提供明确的下一步操作指引
4. **可追踪**：包含唯一告警 ID，便于问题追踪和复盘

## 7.3.2 告警 ID 设计

### 7.3.2.1 ID 格式

```
HB-{timestamp}-{taskId}-{sequence}
```

| 组成部分 | 格式 | 示例 | 说明 |
|---------|------|------|------|
| `HB` | 固定前缀 | `HB` | HeartBeat 缩写 |
| `timestamp` | YYYYMMDDHHmmss | `20240115143000` | 告警发生时间 |
| `taskId` | 任务标识 | `task-001` | 关联任务 ID |
| `sequence` | 3 位序号 | `001` | 同一任务当日第 N 条告警 |

**完整示例**：`HB-20240115143000-task-001-001`

### 7.3.2.2 ID 用途

| 用途 | 说明 |
|------|------|
| **问题追踪** | 在日志、监控系统中通过 ID 定位告警记录 |
| **告警去重** | 防止同一问题重复发送告警 |
| **复盘分析** | 通过 ID 关联告警前后的系统状态 |
| **工单关联** | 告警升级为工单时作为关联标识 |
| **用户反馈** | 用户可通过 ID 反馈告警处理结果 |

### 7.3.2.3 ID 生成实现

```typescript
class AlertIdGenerator {
  private dailyCounters: Map<string, number> = new Map();
  
  generate(taskId: string): string {
    const timestamp = formatDateTime(new Date(), 'YYYYMMDDHHmmss');
    const dateKey = formatDateTime(new Date(), 'YYYYMMDD');
    
    const counterKey = `${dateKey}-${taskId}`;
    const sequence = (this.dailyCounters.get(counterKey) || 0) + 1;
    this.dailyCounters.set(counterKey, sequence);
    
    const paddedSequence = sequence.toString().padStart(3, '0');
    return `HB-${timestamp}-${taskId}-${paddedSequence}`;
  }
  
  parse(alertId: string): AlertIdComponents {
    const match = alertId.match(/^HB-(\d{14})-(.+)-(\d{3})$/);
    if (!match) throw new Error(`Invalid alert ID: ${alertId}`);
    
    return {
      timestamp: parseDateTime(match[1], 'YYYYMMDDHHmmss'),
      taskId: match[2],
      sequence: parseInt(match[3], 10)
    };
  }
}
```

## 7.3.4 发送条件

| 事件类型 | 触发条件 | 发送时机 | 模板 |
|---------|---------|---------|------|
| 任务中断 | 进入 `task-interrupted` 状态 | 立即发送 | 任务中断告警 |
| 恢复耗尽 | 重试次数达到上限 | 立即发送 | 恢复耗尽告警 |
| 权限不足 | 恢复动作被权限系统拒绝 | 立即发送 | 系统错误告警 |
| 检测器异常 | 检测器连续失败 3 次 | 立即发送 | 系统错误告警 |
| DOM 异常 | 关键 DOM 元素无法访问 | 立即发送 | 系统错误告警 |

## 7.3.5 告警升级机制

当 critical 告警在指定时间内未被处理，触发升级：

```
┌─────────────────────────────────────────────────────────────┐
│                    critical 告警发送                         │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
                    ┌─────────────────┐
                    │ 5 分钟内无响应？  │
                    └─────────────────┘
                     │            │
                    是            否
                     │            │
                     ▼            ▼
            ┌──────────────┐  ┌─────────────┐
            │ 升级到值班人员 │  │ 告警已处理   │
            │ 发送短信通知   │  │             │
            └──────────────┘  └─────────────┘
                     │
                     ▼
            ┌─────────────────┐
            │ 15 分钟内无响应？ │
            └─────────────────┘
                     │            │
                    是            否
                     │            │
                     ▼            ▼
            ┌──────────────┐  ┌─────────────┐
            │ 升级到团队负责人 │  │ 告警已处理   │
            │ 电话通知        │  │             │
            └──────────────┘  └─────────────┘
```

## 7.3.6 变量说明

| 变量名 | 类型 | 来源 | 示例值 |
|-------|------|------|--------|
| `alertId` | string | AlertIdGenerator | `HB-20240115143000-task-001-001` |
| `ownerUserId` | string | 任务归属 | `ou_xxx` |
| `ownerName` | string | 任务归属 | `张三` |
| `taskId` | string | HeartbeatDetector | `task-2024-001` |
| `workspace` | string | 配置 | `PMCLI` |
| `interruptReason` | string | 状态机 | `用户手动取消` |
| `lastContent` | string | DOM 提取 | `正在执行 npm install...` |
| `suggestedActions` | string | 预设规则 | `1. 检查网络连接\n2. 重新启动任务` |
| `retryCount` | number | 恢复执行器 | `3` |
| `maxRetry` | number | 配置 | `3` |
| `failureHistory` | string | 恢复执行器 | `- 14:25:01 尝试 stop+retry 失败\n- 14:26:01 尝试 stop+retry 失败` |
| `possibleCauses` | string | 预设规则 | `- AI 服务异常\n- 网络连接不稳定` |
| `errorType` | string | 错误分类 | `DOM_ACCESS_DENIED` |
| `errorDetail` | string | 错误堆栈 | `Error: Cannot access...` |
| `impactScope` | string | 影响分析 | `当前任务无法继续执行` |

## 7.3.7 实现示例

```typescript
interface CriticalAlert {
  alertId: string;
  taskId: string;
  workspace: string;
  eventType: 'task_interrupted' | 'recovery_exhausted' | 'system_error';
  ownerUserId: string;
  ownerName: string;
  interruptReason?: string;
  anomalyType?: HealthMode;
  retryCount?: number;
  maxRetry?: number;
  failureHistory?: string[];
  errorType?: string;
  errorDetail?: string;
  timestamp: Date;
}

function formatCriticalAlert(data: CriticalAlert): LarkCard {
  switch (data.eventType) {
    case 'task_interrupted':
      return buildTaskInterruptedCard(data);
    case 'recovery_exhausted':
      return buildRecoveryExhaustedCard(data);
    case 'system_error':
      return buildSystemErrorCard(data);
  }
}

function shouldSendCritical(event: CriticalEvent): boolean {
  return true;
}
```
