# 7.4 通知频率控制机制

## 7.4.1 设计目标

通知频率控制（Rate Limiting）旨在：

1. **防止通知轰炸**：避免短时间内发送大量重复通知
2. **保证关键信息**：确保 critical 级告警不受限制
3. **提升用户体验**：减少无效打扰，提高通知价值
4. **系统保护**：防止通知系统过载

## 7.4.2 shouldNotify() 核心实现

### 7.4.2.1 函数签名

```typescript
interface NotificationContext {
  level: 'info' | 'warning' | 'critical';
  taskId: string;
  workspace: string;
  eventType: string;
  anomalyType?: HealthMode;
  alertId?: string;
}

interface RateLimitConfig {
  info: {
    windowMs: number;
    maxCount: number;
    aggregate: boolean;
  };
  warning: {
    windowMs: number;
    maxCount: number;
    cooldownMs: number;
  };
  critical: {
    bypass: boolean;
  };
}

function shouldNotify(
  context: NotificationContext,
  config: RateLimitConfig
): boolean;
```

### 7.4.2.2 完整实现

```typescript
class NotificationRateLimiter {
  private notificationHistory: Map<string, NotificationRecord[]> = new Map();
  private lastNotificationTime: Map<string, Date> = new Map();
  
  private config: RateLimitConfig = {
    info: {
      windowMs: 5 * 60 * 1000,
      maxCount: 10,
      aggregate: true
    },
    warning: {
      windowMs: 5 * 60 * 1000,
      maxCount: 3,
      cooldownMs: 5 * 60 * 1000
    },
    critical: {
      bypass: true
    }
  };

  shouldNotify(context: NotificationContext): boolean {
    if (context.level === 'critical') {
      return this.shouldNotifyCritical(context);
    }
    
    if (context.level === 'warning') {
      return this.shouldNotifyWarning(context);
    }
    
    return this.shouldNotifyInfo(context);
  }

  private shouldNotifyCritical(context: NotificationContext): boolean {
    if (!this.config.critical.bypass) {
      return this.checkDuplicate(context);
    }
    return true;
  }

  private shouldNotifyWarning(context: NotificationContext): boolean {
    const key = this.buildKey(context);
    
    if (this.isInCooldown(key)) {
      return false;
    }
    
    const recentCount = this.countRecentNotifications(
      key,
      this.config.warning.windowMs
    );
    
    if (recentCount >= this.config.warning.maxCount) {
      return false;
    }
    
    this.recordNotification(context);
    return true;
  }

  private shouldNotifyInfo(context: NotificationContext): boolean {
    const key = this.buildKey(context);
    
    const recentCount = this.countRecentNotifications(
      key,
      this.config.info.windowMs
    );
    
    if (recentCount >= this.config.info.maxCount) {
      if (this.config.info.aggregate) {
        this.scheduleAggregation(context);
      }
      return false;
    }
    
    this.recordNotification(context);
    return true;
  }

  private buildKey(context: NotificationContext): string {
    return `${context.level}:${context.taskId}:${context.eventType}`;
  }

  private isInCooldown(key: string): boolean {
    const lastTime = this.lastNotificationTime.get(key);
    if (!lastTime) return false;
    
    const elapsed = Date.now() - lastTime.getTime();
    return elapsed < this.config.warning.cooldownMs;
  }

  private countRecentNotifications(key: string, windowMs: number): number {
    const records = this.notificationHistory.get(key) || [];
    const cutoff = Date.now() - windowMs;
    
    return records.filter(r => r.timestamp.getTime() > cutoff).length;
  }

  private recordNotification(context: NotificationContext): void {
    const key = this.buildKey(context);
    const records = this.notificationHistory.get(key) || [];
    
    records.push({
      timestamp: new Date(),
      context
    });
    
    this.notificationHistory.set(key, records);
    this.lastNotificationTime.set(key, new Date());
    
    this.cleanupOldRecords();
  }

  private checkDuplicate(context: NotificationContext): boolean {
    const key = this.buildKey(context);
    const records = this.notificationHistory.get(key) || [];
    const recentThreshold = 60 * 1000;
    
    return !records.some(r => 
      Date.now() - r.timestamp.getTime() < recentThreshold
    );
  }

  private scheduleAggregation(context: NotificationContext): void {
    const aggregationKey = `aggregate:${context.taskId}`;
    const pending = this.pendingAggregations.get(aggregationKey) || [];
    pending.push(context);
    this.pendingAggregations.set(aggregationKey, pending);
  }

  private cleanupOldRecords(): void {
    const maxAge = 30 * 60 * 1000;
    const cutoff = Date.now() - maxAge;
    
    for (const [key, records] of this.notificationHistory) {
      const filtered = records.filter(r => r.timestamp.getTime() > cutoff);
      if (filtered.length === 0) {
        this.notificationHistory.delete(key);
      } else {
        this.notificationHistory.set(key, filtered);
      }
    }
  }
}
```

## 7.4.3 频率控制策略

### 7.4.3.1 info 级策略

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `windowMs` | 5 分钟 | 统计时间窗口 |
| `maxCount` | 10 | 窗口内最大通知数 |
| `aggregate` | true | 超限时是否聚合 |

**行为**：
- 5 分钟内同一任务最多发送 10 条 info 通知
- 超过限制后，后续事件进入聚合队列
- 聚合消息在窗口结束时统一发送

### 7.4.3.2 warning 级策略

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `windowMs` | 5 分钟 | 统计时间窗口 |
| `maxCount` | 3 | 窗口内最大通知数 |
| `cooldownMs` | 5 分钟 | 同类通知冷却时间 |

**行为**：
- 5 分钟内同一类型警告最多发送 3 条
- 同一警告类型发送后进入 5 分钟冷却期
- 冷却期内同类警告被丢弃

### 7.4.3.3 critical 级策略

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `bypass` | true | 是否绕过所有限制 |

**行为**：
- 不受任何频率限制
- 仅做 1 分钟内去重检查
- 确保关键告警即时送达

## 7.4.4 去重机制

### 7.4.4.1 去重 Key 设计

```typescript
function buildDedupKey(context: NotificationContext): string {
  const parts = [
    context.level,
    context.taskId,
    context.eventType
  ];
  
  if (context.anomalyType) {
    parts.push(context.anomalyType);
  }
  
  if (context.alertId) {
    parts.push(context.alertId);
  }
  
  return parts.join(':');
}
```

### 7.4.4.2 去重规则

| 级别 | 去重窗口 | 去重维度 |
|------|---------|---------|
| `info` | 1 分钟 | taskId + eventType |
| `warning` | 5 分钟 | taskId + eventType + anomalyType |
| `critical` | 1 分钟 | alertId |

## 7.4.5 聚合机制

### 7.4.5.1 聚合触发条件

```typescript
interface AggregationTrigger {
  type: 'count_exceeded' | 'time_window' | 'manual';
  pendingNotifications: NotificationContext[];
}

function shouldTriggerAggregation(
  key: string,
  config: RateLimitConfig
): AggregationTrigger | null {
  const pending = this.pendingAggregations.get(key) || [];
  
  if (pending.length >= 5) {
    return { type: 'count_exceeded', pendingNotifications: pending };
  }
  
  const oldest = pending[0];
  if (oldest && Date.now() - oldest.timestamp.getTime() > config.info.windowMs) {
    return { type: 'time_window', pendingNotifications: pending };
  }
  
  return null;
}
```

### 7.4.5.2 聚合消息生成

```typescript
function buildAggregatedMessage(
  notifications: NotificationContext[]
): AggregatedNotification {
  const grouped = groupBy(notifications, 'eventType');
  
  return {
    taskId: notifications[0].taskId,
    workspace: notifications[0].workspace,
    timeWindow: '5 分钟',
    summary: Object.entries(grouped).map(([eventType, items]) => ({
      eventType,
      count: items.length,
      firstOccurrence: minBy(items, 'timestamp'),
      lastOccurrence: maxBy(items, 'timestamp')
    })),
    timestamp: new Date()
  };
}
```

### 7.4.6.2 动态配置

```typescript
interface RateLimitConfigUpdate {
  level: 'info' | 'warning' | 'critical';
  config: Partial<RateLimitConfig['info' | 'warning' | 'critical']>;
}

function updateRateLimitConfig(update: RateLimitConfigUpdate): void {
  const current = this.config[update.level];
  this.config[update.level] = { ...current, ...update.config };
}
```

### 7.4.6.3 场景化配置

| 场景 | info.maxCount | warning.maxCount | 说明 |
|------|---------------|------------------|------|
| 正常模式 | 10 | 3 | 默认配置 |
| 高频模式 | 20 | 5 | 大型任务，预期有较多异常 |
| 安静模式 | 5 | 1 | 减少打扰，仅关键通知 |
| 调试模式 | 50 | 20 | 开发调试，详细日志 |
