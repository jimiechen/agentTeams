# 7.2 info 级消息格式

## 7.2.1 设计原则

info 级消息作为"心跳信号"，设计遵循以下原则：

1. **简洁性**：一眼扫过即可理解核心信息
2. **可聚合性**：支持多条消息合并展示
3. **低打扰**：不@用户，不触发强提醒
4. **可追溯**：包含任务标识和时间戳

## 7.2.3 发送条件

| 事件类型 | 触发条件 | 发送时机 | 模板 |
|---------|---------|---------|------|
| 恢复成功 | `terminal-hang` → `healthy` | 动作执行后立即发送 | 恢复成功通知 |
| 恢复成功 | `modal-blocking` → `healthy` | 动作执行后立即发送 | 恢复成功通知 |
| 恢复成功 | `model-stalled` → `healthy` | 验证窗口确认恢复后发送 | 恢复成功通知 |
| 任务完成 | `completed` 状态 | 任务进入终态时发送 | 任务完成通知 |
| 排队开始 | 进入 `model-queuing` | 检测到排队提示时发送 | 排队状态通知 |
| 排队结束 | 退出 `model-queuing` | AI 开始响应时发送 | 排队状态通知（位置=0） |

## 7.2.4 消息聚合策略

当短时间内发生多个 info 级事件时，采用聚合策略减少消息数量：

### 7.2.4.1 聚合条件

- 时间窗口：5 分钟
- 聚合维度：同一任务、同一工作区
- 聚合事件：恢复成功、排队状态变化

### 7.2.4.2 聚合消息模板

```json
{
  "msg_type": "interactive",
  "card": {
    "header": {
      "title": {
        "tag": "plain_text",
        "content": "📊 任务运行摘要"
      },
      "template": "green"
    },
    "elements": [
      {
        "tag": "div",
        "text": {
          "tag": "lark_md",
          "content": "**任务 ${taskId}** 在过去 ${timeWindow} 内："
        }
      },
      {
        "tag": "div",
        "text": {
          "tag": "lark_md",
          "content": "- ✅ 自动处理终端阻塞 ${terminalHangCount} 次\n- ✅ 自动处理弹窗 ${modalBlockingCount} 次\n- ⏳ 排队等待 ${queuingTime} 分钟"
        }
      },
      {
        "tag": "note",
        "elements": [
          {
            "tag": "plain_text",
            "content": "⏰ ${timestamp}"
          }
        ]
      }
    ]
  }
}
```

## 7.2.5 变量说明

| 变量名 | 类型 | 来源 | 示例值 |
|-------|------|------|--------|
| `taskId` | string | HeartbeatDetector | `task-2024-001` |
| `workspace` | string | 配置 | `PMCLI` / `DEVCLI` |
| `anomalyType` | string | 健康模式 | `terminal-hang` |
| `recoveryTime` | number | 恢复耗时统计 | `1500` |
| `timestamp` | string | 当前时间 | `2024-01-15 14:30:00` |
| `totalDuration` | string | 任务总耗时 | `2h 15m` |
| `taskSummary` | string | 任务摘要 | `完成用户认证模块开发` |
| `taskDetailUrl` | string | 任务详情页 URL | `https://...` |
| `queuePosition` | number | 排队位置 | `3` |
| `estimatedWaitTime` | string | 预计等待时间 | `约 2 分钟` |

## 7.2.6 实现示例

```typescript
interface InfoNotification {
  taskId: string;
  workspace: string;
  eventType: 'recovery_success' | 'task_completed' | 'queuing';
  anomalyType?: HealthMode;
  recoveryTime?: number;
  queuePosition?: number;
  timestamp: Date;
}

function formatInfoNotification(data: InfoNotification): LarkCard {
  switch (data.eventType) {
    case 'recovery_success':
      return buildRecoverySuccessCard(data);
    case 'task_completed':
      return buildTaskCompletedCard(data);
    case 'queuing':
      return buildQueuingCard(data);
  }
}

function shouldSendInfo(event: InfoEvent): boolean {
  if (event.type === 'recovery_success') {
    return event.anomalyType !== 'model-queuing';
  }
  return true;
}
```
